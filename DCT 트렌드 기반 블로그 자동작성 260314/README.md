# DCT 트렌드 기반 블로그 자동작성 워크플로우

본 워크플로우는 매주 일요일 오전 9시에 네이버 쇼핑 트렌드를 분석하여, 현재 시기에 최적화된 판촉물, 홍보물, 답례품 추천 블로그 포스팅을 자동으로 생성하고 노션(Notion) 데이터베이스에 저장하는 자동화 시스템입니다.

---

## 📋 개요
- **목적**: 시즌별 트렌드 데이터를 기반으로 한 고품질 블로그 콘텐츠 자동 생성
- **실행 주기**: 매주 일요일 오전 9시 (Schedule Trigger)
- **핵심 기술**: n8n, 네이버 데이터랩 API, Google Gemini AI, Notion API

---

## 🔄 워크플로우 흐름
1. **Schedule Trigger**: 매주 일요일 오전 9시 자동 실행
2. **현재 시기 정보**: 날짜를 기반으로 계절, 이벤트, API 요청용 날짜 범위 계산
3. **네이버 트렌드 수집**: 네이버 데이터랩 API를 통해 판촉/생활용품/답례품 카테고리 검색량 분석
4. **데이터 통합**: 시기 정보와 트렌드 데이터를 결합하여 프롬프트 생성
5. **Gemini 블로그 작성**: Gemini AI가 SEO를 고려한 블로그 초안 작성
6. **블로그 추출 및 파싱**: 응답 데이터 정제 및 노션 저장용 청크 분할
7. **노션 저장**: 최종 결과물을 노션 DB에 페이지로 생성

---

## 🧩 노드 상세 설명
- **현재 시기 정보 (Code)**: 계절별(봄/여름/가을/겨울) 특수 이벤트와 네이버 API용 날짜를 동적으로 생성합니다.
- **네이버 트렌드 수집 (HTTP Request)**: `X-Naver-Client-Id`와 `Secret`을 사용하여 최신 검색 트렌드를 가져옵니다.
- **Gemini 블로그 작성 (HTTP Request)**: `gemini-3.1-flash-lite-preview` 모델을 사용하여 블로그 포스팅을 생성합니다. (환경변수 `GEMINI_API_KEY` 필요)
- **노션 저장 (HTTP Request)**: 생성된 콘텐츠를 지정된 노션 데이터베이스(`39b2fdcea0a345a48db21f22a6d6b0e0`)에 업로드합니다.

## 🔑 필수 설정 및 환경변수
워크플로우 정상 작동을 위해 다음 설정이 필요합니다.

1. **환경변수**:
   - `GEMINI_API_KEY`: Google AI Studio에서 발급받은 API 키를 n8n 환경변수로 설정해야 합니다.
2. **API 인증**:
   - **네이버**: `X-Naver-Client-Id`, `X-Naver-Client-Secret`을 노드 헤더에 입력하십시오.
   - **노션**: 노션 DB의 `Connections` 설정에서 해당 워크플로우가 사용하는 인테그레이션을 반드시 연결해야 합니다.

---

## 🛠️ 트러블슈팅
- **401 인증 오류**: 네이버 개발자 센터에서 API 키가 유효한지 확인하세요.
- **429 API 한도 초과**: Gemini API 사용량이 초과된 경우입니다. 잠시 후 재실행하거나 사용량을 확인하세요.
- **404 노션 오류**: 노션 DB에 인테그레이션이 연결되어 있는지 확인하세요.
- **400 노션 본문 길이 제한**: 노션은 블록당 2000자 제한이 있습니다. 본문이 길 경우 '블로그 추출' 노드에서 청크 분할 로직을 조정하세요.

# 📚 Math4U - 수업 리포트 자동화 (Thinking Flow)

본 워크플로우는 대치 Math4U 학원의 수업 후 리포트 작성 및 발송 과정을 자동화하여, 강사의 업무 효율을 높이고 학부모에게는 프리미엄 수준의 맞춤형 수업 피드백을 제공하기 위해 설계되었습니다.

## 🎯 목적
강사가 모바일 폼(Google Forms/Tally)을 통해 입력한 수업 데이터를 기반으로, AI(Gemini)가 전문적인 'The Thinking Flow(사고의 궤적)' 리포트를 생성하고 이를 학부모에게 카카오알림톡으로 자동 발송합니다.

## 🔄 워크플로우 흐름
1. **Webhook 수신**: 모바일 폼으로부터 수업 데이터(학생명, 사고력 지표, 유레카 모먼트 등)를 실시간으로 수신합니다.
2. **데이터 파싱**: 수신된 데이터를 표준화된 포맷으로 변환하여 이후 노드에서 원활하게 사용할 수 있도록 정리합니다.
3. **Notion 학생 조회**: Notion 데이터베이스에서 해당 학생의 정보 및 학부모 연락처를 조회합니다.
4. **AI 리포트 생성**: Gemini API를 활용하여 입력된 데이터를 바탕으로 '에르메스급 품격'의 전문적인 수업 리포트를 생성합니다.
5. **알림톡 발송**: 생성된 리포트를 학부모에게 카카오알림톡으로 발송합니다.
6. **Notion 기록**: 발송된 리포트 내용을 Notion DB에 기록하여 이력을 관리합니다.
7. **Slack 알림**: 강사(카이)에게 발송 완료 여부를 Slack으로 알림 전송합니다.

## ⚙️ 필수 설정 및 환경변수
워크플로우를 정상적으로 실행하기 위해 다음 항목이 필요합니다.

*   **환경변수 (Environment Variables)**:
    *   `GEMINI_API_KEY`: AI 리포트 생성을 위한 Google Gemini API 키
*   **크레덴셜 (Credentials)**:
    *   **Notion API**: 학생 데이터 조회 및 기록을 위한 연동
    *   **HTTP Request**: 카카오알림톡 발송을 위한 API 연동
    *   **Slack**: 발송 완료 알림을 위한 Webhook 또는 API 연동

## 📋 모바일 폼 연동 방법
1. Google Forms 또는 Tally 등 선호하는 폼 빌더를 사용합니다.
2. 폼 제출 시 데이터를 전송할 **Webhook URL**을 본 n8n 워크플로우의 `Webhook` 노드에서 복사하여 설정합니다.
3. 폼의 필드명은 워크플로우의 `데이터 파싱` 노드에서 정의한 키값(`studentName`, `lessonDate`, `subjectUnit` 등)과 일치시켜야 합니다.

## ⚠️ 주의사항
*   **AI 프롬프트**: 리포트의 톤앤매너를 유지하기 위해 `AI 리포트 생성` 노드의 프롬프트는 수정 시 주의가 필요합니다.
*   **데이터 필터링**: `Notion 학생 조회` 노드에서 학생 이름이 정확히 매칭되지 않을 경우 리포트 발송이 실패할 수 있으므로, Notion DB의 데이터 정합성을 유지해 주세요.
*   **알림톡 글자 수**: 카카오알림톡의 글자 수 제한(1,000자)을 고려하여 프롬프트가 설계되어 있습니다. 리포트 내용이 너무 길어지지 않도록 주의하십시오.

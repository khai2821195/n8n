# 📚 Math4U - 수업 리포트 자동화 (Thinking Flow)

본 워크플로우는 대치 Math4U 수학학원의 수업 후 리포트 작성 및 발송 과정을 자동화하여, 강사의 업무 효율을 높이고 학부모에게 고품질의 'The Thinking Flow(사고의 궤적)' 리포트를 제공하기 위해 설계되었습니다.

## 🎯 워크플로우 개요
강사가 모바일 폼(Google Forms/Tally 등)을 통해 수업 내용을 제출하면, n8n이 이를 받아 AI(Gemini)를 통해 전문적인 리포트로 변환하고, 학부모에게 카카오알림톡으로 발송한 뒤 노션(Notion)에 기록하는 자동화 시스템입니다.

## 🔄 워크플로우 단계
1. **📱 폼 데이터 수신 (Webhook)**: 외부 폼으로부터 수업 데이터를 POST 방식으로 수신합니다.
2. **📊 데이터 파싱**: 수신된 데이터를 표준화하여 학생 이름, 수업 단원, 사고력 지표(Discovery, Generalization, Concentration) 등을 추출합니다.
3. **🔍 Notion 학생 조회**: 노션 데이터베이스에서 해당 학생의 정보를 조회하여 학부모 연락처 및 기본 데이터를 확보합니다.
4. **🤖 AI 리포트 생성**: Gemini API를 호출하여 입력된 데이터를 바탕으로 '대치 Math4U 프리미엄' 톤앤매너에 맞춘 수업 리포트를 자동 생성합니다.
5. **💬 알림톡 발송**: 생성된 리포트를 학부모에게 카카오알림톡으로 전송합니다.
6. **📝 Notion 기록**: 발송된 리포트 내용을 노션 데이터베이스에 저장하여 이력을 관리합니다.
7. **🔔 Slack 알림**: 강사(카이)에게 최종 발송 완료 알림을 전송합니다.

## ⚙️ 필수 설정 및 환경 변수
워크플로우를 정상적으로 실행하기 위해 다음 설정이 필요합니다.

*   **환경 변수**:
    *   `GEMINI_API_KEY`: AI 리포트 생성을 위한 Google Gemini API 키
*   **크레덴셜(Credentials)**:
    *   **Notion API**: 학생 정보 조회 및 리포트 기록을 위한 연동 권한
    *   **Kakao Alimtalk**: 알림톡 발송을 위한 API 연동
    *   **Slack**: 발송 완료 알림을 위한 Webhook 또는 API 연동
*   **데이터베이스**:
    *   노션 내 학생 정보 및 리포트 기록을 위한 데이터베이스 ID가 `notion-student-lookup` 노드에 정확히 설정되어 있어야 합니다.

## ⚠️ 주의사항
*   **Webhook URL**: 모바일 폼(Google Forms/Tally)의 제출 완료 후 호출되는 Webhook URL이 본 워크플로우의 `webhook-trigger` 노드 경로(`math4u-lesson-report`)와 일치하는지 확인하십시오.
*   **데이터 매핑**: 노션 데이터베이스의 속성명(예: `학생이름|title`)이 실제 노션 DB의 컬럼명과 일치해야 데이터 조회가 가능합니다.
*   **AI 프롬프트**: `ai-report-generate` 노드의 프롬프트는 학원 브랜드 가이드라인에 따라 수정 가능합니다.

## 💡 활용 팁
*   폼 제출 시 `presetMessage` 필드를 활용하면, 강사가 특정 수업에서 강조하고 싶은 내용을 추가로 입력하여 AI 리포트에 반영할 수 있습니다.
*   테스트 시에는 실제 학부모 연락처가 아닌 테스트용 번호를 사용하여 알림톡 발송 여부를 먼저 확인하는 것을 권장합니다.

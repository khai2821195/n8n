# 📚 Math4U - 수업 리포트 자동화 (Thinking Flow)

본 워크플로우는 대치 Math4U 수학학원의 수업 리포트 작성 및 발송 과정을 자동화하여, 강사의 업무 효율을 높이고 학부모에게 고품질의 'The Thinking Flow(사고의 궤적)' 리포트를 제공하기 위해 설계되었습니다.

## 🎯 워크플로우 목적
강사가 모바일 폼(Google Forms/Tally 등)을 통해 수업 내용을 제출하면, n8n이 이를 분석하여 AI(Gemini)를 통해 전문적인 리포트를 생성하고, 학부모에게 카카오알림톡으로 즉시 발송하며, 노션(Notion)에 이력을 기록합니다.

## 🔄 워크플로우 단계
1. **📱 폼 데이터 수신 (Webhook)**: 모바일 폼으로부터 수업 데이터를 POST 방식으로 수신합니다.
2. **📊 데이터 파싱**: 수신된 JSON 데이터를 워크플로우에서 사용하기 적합한 형식으로 정제합니다.
3. **🔍 Notion 학생 조회**: 입력된 학생 이름을 바탕으로 노션 DB에서 학부모 연락처 및 학생 정보를 조회합니다.
4. **🤖 AI 리포트 생성**: Gemini API를 호출하여 수업 데이터를 바탕으로 '대치 Math4U 프리미엄' 톤앤매너에 맞춘 리포트 문장을 생성합니다.
5. **💬 알림톡 발송**: 생성된 리포트를 학부모에게 카카오알림톡으로 발송합니다.
6. **📝 Notion 기록**: 발송된 리포트 내용을 노션 DB에 자동으로 저장하여 관리합니다.
7. **🔔 Slack 알림**: 강사(카이)에게 발송 완료 여부를 슬랙으로 통보합니다.

## ⚙️ 환경 설정 및 요구사항

### 1. 필수 환경변수 (Environment Variables)
n8n 서버 설정에 다음 환경변수가 등록되어 있어야 합니다.
- `GEMINI_API_KEY`: AI 리포트 생성을 위한 Google Gemini API 키

### 2. 크레덴셜 (Credentials)
- **Notion API**: 학생 정보 조회 및 리포트 기록을 위한 연동 권한 필요.
- **Webhook**: 외부 폼 서비스(Google Forms 등)에서 해당 Webhook URL로 데이터를 보낼 수 있도록 설정.
- **알림톡/Slack**: 각 서비스 연동을 위한 API 토큰 및 설정.

## 🛠 사용 방법
1. **Webhook URL 설정**: 워크플로우의 `📱 폼 데이터 수신` 노드에서 생성된 Webhook URL을 복사하여 모바일 폼의 제출 목적지로 설정합니다.
2. **데이터 매핑**: 폼의 필드명과 워크플로우의 파싱 노드 필드명이 일치하는지 확인합니다.
3. **Notion DB 확인**: 학생 정보가 담긴 노션 데이터베이스의 속성명(`학생이름|title` 등)이 워크플로우의 필터 조건과 일치하는지 확인합니다.

## ⚠️ 주의사항
- **데이터 형식**: 폼 제출 시 `studentName` 필드는 노션 DB의 학생 이름과 정확히 일치해야 합니다.
- **AI 모델**: 현재 `gemini-3.1-flash-lite` 모델을 사용 중입니다. 필요에 따라 모델명이나 프롬프트를 수정하여 리포트의 톤앤매너를 조정할 수 있습니다.
- **에러 핸들링**: 노션에서 학생 정보를 찾지 못할 경우를 대비하여 예외 처리 로직을 추가하는 것을 권장합니다.

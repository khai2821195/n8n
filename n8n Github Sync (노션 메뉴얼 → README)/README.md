# n8n Github Sync (노션 메뉴얼 → README)

이 워크플로우는 n8n에서 관리되는 워크플로우의 상태를 노션(Notion) 데이터베이스와 동기화하고, 각 워크플로우의 정의(JSON)와 문서(README.md)를 GitHub 저장소에 자동으로 백업 및 업데이트하는 자동화 시스템입니다.

## 🔄 워크플로우 동작 흐름

1. **수동 실행**: 사용자가 워크플로우를 수동으로 트리거합니다.
2. **n8n 전체 목록 조회**: n8n API를 통해 현재 시스템에 등록된 모든 워크플로우 목록을 가져옵니다.
3. **노션 DB 조회**: 동기화 대상인 노션 데이터베이스의 항목들을 불러옵니다.
4. **동기화 비교 (Code 노드)**:
   - n8n에는 없거나 아카이브된 워크플로우를 노션에서 삭제(아카이브) 처리합니다.
   - 노션에 등록되지 않은 신규 활성 워크플로우를 식별합니다.
5. **작업 분류 및 처리**:
   - **삭제 작업**: 노션 페이지를 업데이트하여 아카이브 처리합니다.
   - **추가/업데이트 작업**: 
     - n8n API를 통해 `workflow.json`을 추출합니다.
     - 노션에 작성된 메뉴얼 내용을 마크다운으로 변환합니다.
     - 최종적으로 `workflow.json`과 `README.md`를 지정된 GitHub 저장소에 커밋합니다.

## 📁 GitHub 저장소 구조

```text
workflows/
  워크플로우이름/
    workflow.json
    README.md
```

## 🔧 필수 환경변수 및 크레덴셜

워크플로우가 정상적으로 동작하기 위해 다음 환경변수와 API 키 설정이 필요합니다.

*   **`N8N_API_KEY`**: n8n API 접근을 위한 인증 키
*   **`GITHUB_TOKEN`**: GitHub 저장소에 파일을 생성/수정하기 위한 Personal Access Token
*   **`GEMINI_API_KEY`**: 노션 메뉴얼을 마크다운으로 변환하거나 README를 생성할 때 사용하는 AI 모델 키
*   **`NOTION_API_KEY`**: 노션 데이터베이스 조회 및 페이지 수정을 위한 인증 토큰

## ⚠️ 주의사항 및 유지보수

*   **GitHub 연동**: 저장소 경로가 변경될 경우, GitHub API 호출 노드(JSON 저장 및 README 저장)의 URL 설정을 업데이트해야 합니다.
*   **노션 DB 변경**: 데이터베이스 ID가 변경될 경우, '노션 DB 조회' 노드의 URL 내 DB ID 값을 수정하십시오.
*   **신규 워크플로우 추가**: 노션 데이터베이스에 새로운 항목을 추가할 때, 반드시 `Workflow ID` 컬럼에 해당 n8n 워크플로우의 고유 ID를 정확히 입력해야 동기화가 진행됩니다.
*   **에러 처리**: 워크플로우 실행 중 특정 노드에서 오류가 발생할 경우, '노션 아카이브 WF 삭제' 노드와 같이 `continueRegularOutput` 옵션이 설정된 노드들의 동작을 확인하십시오.

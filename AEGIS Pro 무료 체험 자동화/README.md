# AEGIS Pro 무료 체험 자동화 워크플로우

본 워크플로우는 사용자가 노션(Notion) 데이터베이스를 통해 AEGIS Pro 무료 체험을 신청하면, 이를 자동으로 감지하여 Supabase를 통해 Pro 기능을 활성화하고 노션 상태를 업데이트하는 자동화 시스템입니다.

## 📋 워크플로우 개요
- **목적**: AEGIS Pro 무료 체험 신청 프로세스 자동화
- **트리거**: 노션 데이터베이스 신규 항목 감지
- **주요 기능**:
    - Supabase Edge Function 호출을 통한 Pro 권한 활성화
    - 결과에 따른 노션 데이터베이스 상태값 자동 업데이트 (성공/오류)
    - Pro 시작일 및 만료일(30일) 자동 계산 및 기록

## 🔄 워크플로우 흐름
1. **노션 신청 감지 (Notion Trigger)**: 지정된 노션 데이터베이스를 1분마다 모니터링하여 새로운 신청 건을 감지합니다.
2. **Supabase Pro 활성화 (HTTP Request)**: 신청자의 정보(ID, 이메일, 이름)를 Supabase Edge Function(`activate-pro-trial`)으로 전송하여 Pro 권한을 부여합니다.
3. **성공 여부 확인 (If)**: API 응답 결과(`success`)를 확인하여 분기 처리합니다.
4. **노션 상태 업데이트**:
    - **성공 시**: 상태를 'Pro 적용됨'으로 변경하고, 시작일과 30일 후의 만료일을 기록합니다.
    - **오류 시**: 상태를 '오류'로 변경하고, 상세 오류 메시지를 메모란에 기록합니다.

## ⚙️ 설정 가이드

### 필수 크레덴셜
- **Notion API**: 데이터베이스 읽기/쓰기 권한이 포함된 API 토큰이 필요합니다.
- **Supabase**: HTTP Request 노드에서 사용할 `x-webhook-secret` 헤더 값이 설정되어 있어야 합니다.

### 환경 변수 및 파라미터
- **Database ID**: 노션 데이터베이스의 고유 ID를 `노션 신청 감지` 노드에 설정해야 합니다.
- **Webhook Secret**: `aegis-pro-trial-2026` 값이 Supabase 함수 측의 인증 값과 일치해야 합니다.

## ⚠️ 주의사항
- **데이터베이스 속성**: 노션 데이터베이스에 아래 속성들이 정확히 존재해야 합니다:
    - `AEGIS ID` (텍스트)
    - `이메일` (텍스트)
    - `이름` (텍스트)
    - `상태` (선택 항목: 'Pro 적용됨', '오류')
    - `Pro 시작일` (날짜)
    - `Pro 만료일` (날짜)
    - `메모` (텍스트)
- **실행 주기**: 현재 1분 단위로 폴링(Polling)하도록 설정되어 있습니다. 트래픽이 많을 경우 n8n 실행 비용을 고려하여 주기를 조정하십시오.

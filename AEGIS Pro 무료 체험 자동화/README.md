# AEGIS Pro 무료 체험 자동화 워크플로우

본 워크플로우는 사용자가 노션(Notion) 데이터베이스를 통해 AEGIS Pro 무료 체험을 신청하면, 이를 자동으로 감지하여 Supabase를 통해 Pro 기능을 활성화하고 노션 상태를 업데이트하는 자동화 시스템입니다.

## 📋 워크플로우 개요
- **목적**: 수동 작업 없이 노션 신청 데이터를 기반으로 Pro 체험 권한을 즉시 부여하고 관리합니다.
- **주요 기능**:
    - 노션 데이터베이스 실시간 모니터링
    - Supabase Edge Function을 통한 Pro 권한 활성화
    - 처리 결과에 따른 노션 상태 자동 업데이트 (성공/오류)

## 🔄 워크플로우 흐름
1. **노션 신청 감지 (Trigger)**: 1분마다 노션 데이터베이스를 폴링하여 새로운 신청 건을 감지합니다.
2. **Supabase Pro 활성화 + 알림**: 신청자의 정보(ID, 이메일, 이름)를 Supabase Edge Function으로 전송하여 Pro 권한을 활성화합니다.
3. **성공 여부 확인**: API 응답 결과를 분석하여 성공 여부를 판단합니다.
4. **노션 상태 업데이트**:
    - **성공 시**: 상태를 'Pro 적용됨'으로 변경하고, 시작일과 30일 후의 만료일을 자동으로 기록합니다.
    - **오류 시**: 상태를 '오류'로 변경하고, 상세 오류 메시지를 메모란에 기록하여 관리자가 확인할 수 있게 합니다.

## ⚙️ 설정 및 요구사항

### 필수 크레덴셜
- **Notion API**: 데이터베이스 읽기 및 페이지 업데이트 권한이 있는 API 토큰이 필요합니다.
- **Supabase**: Edge Function 호출을 위한 권한 설정이 필요합니다.

### 환경 변수 및 파라미터
- **Supabase Webhook Secret**: `x-webhook-secret` 헤더 값으로 `aegis-pro-trial-2026`이 설정되어 있습니다.
- **노션 데이터베이스 ID**: `b208216d-652d-4f89-bf63-7c421b05a4b8` (해당 ID를 사용 중인 데이터베이스 ID로 확인하세요).

## ⚠️ 주의사항
- **데이터베이스 스키마**: 노션 데이터베이스에 다음 필드가 반드시 존재해야 합니다:
    - `상태` (Select 타입: 'Pro 적용됨', '오류' 옵션 포함)
    - `AEGIS ID`, `이메일`, `이름` (Text 타입)
    - `Pro 시작일`, `Pro 만료일` (Date 타입)
    - `메모` (Rich Text 타입)
- **폴링 간격**: 현재 1분 단위로 설정되어 있습니다. 트래픽 상황에 따라 조정이 가능합니다.
- **보안**: Supabase Function 호출 시 사용되는 `x-webhook-secret`은 외부 노출되지 않도록 주의하십시오.

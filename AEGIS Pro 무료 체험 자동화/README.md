# AEGIS Pro 무료 체험 자동화 워크플로우

본 워크플로우는 사용자가 노션(Notion) 데이터베이스를 통해 AEGIS Pro 무료 체험을 신청하면, 이를 자동으로 감지하여 Supabase를 통해 Pro 권한을 활성화하고 노션 상태를 업데이트하는 자동화 시스템입니다.

## 📋 워크플로우 개요
- **목적**: 수동 작업 없이 노션 신청 데이터를 기반으로 Pro 체험 권한 자동 부여 및 관리
- **주요 기능**:
    - 노션 데이터베이스 실시간 모니터링
    - Supabase Edge Function을 통한 Pro 권한 활성화
    - 처리 결과에 따른 노션 데이터베이스 상태 자동 업데이트 (성공/실패)

## 🔄 워크플로우 흐름
1. **노션 신청 감지 (Notion Trigger)**: 지정된 노션 데이터베이스에 새로운 신청 데이터가 들어오는지 1분마다 확인합니다.
2. **Supabase Pro 활성화 (HTTP Request)**: 신청자의 정보(ID, 이메일, 이름)를 Supabase Edge Function(`activate-pro-trial`)으로 전송하여 권한을 활성화합니다.
3. **성공 여부 확인 (If)**: API 응답 결과(`success`)를 확인하여 분기 처리합니다.
4. **노션 상태 업데이트**:
    - **성공 시**: 상태를 'Pro 적용됨'으로 변경하고, 시작일과 30일 후의 만료일을 자동으로 기록합니다.
    - **오류 시**: 상태를 '오류'로 변경하고, 상세 에러 메시지를 메모란에 기록하여 관리자가 확인할 수 있게 합니다.

## ⚙️ 필수 설정 및 환경 요구사항
이 워크플로우를 정상적으로 작동시키기 위해 다음 설정이 필요합니다.

### 1. 크레덴셜 (Credentials)
- **Notion API**: 데이터베이스 접근 권한이 있는 API 토큰이 필요합니다.
- **Supabase**: HTTP 요청 시 사용되는 `x-webhook-secret` 헤더 값이 Supabase Edge Function의 보안 설정과 일치해야 합니다.

### 2. 노션 데이터베이스 구조
데이터베이스에는 다음 필드가 포함되어 있어야 합니다:
- `AEGIS ID` (텍스트)
- `이메일` (텍스트)
- `이름` (텍스트)
- `상태` (선택 항목: 'Pro 적용됨', '오류' 등)
- `Pro 시작일` (날짜)
- `Pro 만료일` (날짜)
- `메모` (텍스트)

## ⚠️ 주의사항
- **Webhook Secret**: Supabase 함수 호출 시 사용되는 `aegis-pro-trial-2026` 시크릿 키가 변경될 경우, 워크플로우의 HTTP Request 노드 헤더 값도 함께 업데이트해야 합니다.
- **데이터 형식**: 노션의 필드명과 워크플로우 내 노드에서 참조하는 필드명이 일치해야 합니다. 필드명 변경 시 노드 설정을 확인하십시오.
- **실행 주기**: 현재 1분 단위로 폴링(Polling)하도록 설정되어 있으므로, 노션 API 호출 제한(Rate Limit)을 고려하여 운영하십시오.

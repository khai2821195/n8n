# 🔧 슬랙 나에게 보내기 (공통 서브워크플로우)

이 워크플로우는 n8n 내의 다른 워크플로우에서 호출하여 슬랙(Slack)으로 메시지를 간편하게 전송할 수 있도록 설계된 **공통 서브워크플로우**입니다.

## 📋 목적
다양한 워크플로우에서 발생하는 알림이나 로그를 중앙화된 방식으로 본인의 슬랙 채널로 전송하기 위해 사용합니다. 매번 슬랙 노드를 설정할 필요 없이, 이 워크플로우를 호출하는 것만으로 메시지 발송이 가능합니다.

## ⚙️ 동작 방식
워크플로우는 다음과 같은 순서로 동작합니다:

1.  **Execute Workflow Trigger**: 상위 워크플로우로부터 `text`(메시지 내용)와 `channel`(선택 사항)을 입력받습니다.
2.  **Code - 채널 설정**: 입력받은 데이터를 검증합니다. `channel` 값이 없으면 기본 설정된 `DEFAULT_CHANNEL`을 사용합니다.
3.  **Slack - 나에게 보내기**: 설정된 채널로 메시지를 전송합니다.
4.  **Code - 결과 정리**: 슬랙 API의 응답을 분석하여 성공 여부(`isSuccess`)와 채널 정보를 반환합니다.

## 📥 입력 파라미터
상위 워크플로우에서 `Execute Workflow` 노드를 통해 다음 데이터를 전달해야 합니다.

| 필드 | 타입 | 필수 여부 | 설명 |
| :--- | :--- | :--- | :--- |
| `text` | string | 필수 | 슬랙으로 보낼 메시지 내용 |
| `channel` | string | 선택 | 메시지를 받을 채널 ID 또는 유저 ID |

## 🛠 설정 방법
1.  **기본 채널 변경**: `Code - 채널 설정` 노드를 엽니다.
2.  `DEFAULT_CHANNEL` 변수에 본인의 슬랙 유저 ID를 입력합니다.
    *   *확인 방법: 슬랙 프로필 → ⋯ (더보기) → Copy member ID*
3.  **크레덴셜**: `Slack - 나에게 보내기` 노드에 올바른 Slack API 토큰(Bot Token)이 연결되어 있는지 확인하세요.

## 📤 출력 결과
호출한 워크플로우는 다음과 같은 JSON 형태의 결과를 받습니다.
```json
{
  "isSuccess": true,
  "channel": "U12345678",
  "ts": "1711111111.000000"
}
```

## ⚠️ 주의사항
*   이 워크플로우는 **서브워크플로우** 전용입니다. 직접 실행하기보다는 `Execute Workflow` 노드를 통해 호출하여 사용하세요.
*   슬랙 앱(Slack App)이 해당 채널에 초대되어 있어야 메시지 전송이 가능합니다. (개인 DM의 경우 앱을 DM 대화방에 추가하거나, 앱 권한을 확인하세요.)

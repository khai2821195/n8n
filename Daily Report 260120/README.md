# Daily Report 260120 (Math4U)

본 워크플로우는 Math4U 수업 관리 시스템의 핵심 자동화 프로세스로, 노션(Notion)에 기록된 수업 데이터를 기반으로 학부모에게 수업 리포트를 발송하고, 특정 조건 충족 시 수업료 납부 알림을 자동으로 전송합니다.

---

## 🔄 워크플로우 흐름

1.  **Webhook**: 노션 데이터 변경 또는 특정 이벤트 발생 시 데이터를 수신합니다.
2.  **Notion - Student_Study 재조회**: 트리거된 페이지의 상세 정보를 가져옵니다.
3.  **Notion - Students 조회**: 해당 수업과 연결된 학생 정보를 조회합니다.
4.  **Notion - Class_Study 조회**: 수업 세부 정보 및 연관된 수업 데이터를 조회합니다.
5.  **Notion - 지난수업 조회**: 이전 수업의 과제 및 진도 정보를 가져와 리포트에 포함합니다.
6.  **Code**: 수신된 데이터를 가공하여 알림톡 발송을 위한 파라미터를 생성합니다.
    *   수업 리포트 데이터 구성
    *   납부 알림 조건 체크 (수업 회차와 수업 구분 번호 비교)
7.  **HTTP Request - 수업 리포트**: 가공된 데이터를 바탕으로 학부모에게 수업 리포트 알림톡을 발송합니다.
8.  **If - 납부 알림 조건 체크**: 수업료 납부 대상 여부를 판단합니다.
    *   **True**: **HTTP Request - 수업료 납부 알림**을 통해 안내 메시지를 발송하고, 노션에 발송 완료 상태를 업데이트합니다.
    *   **False**: 별도의 추가 작업 없이 종료합니다.

---

## ⚙️ 주요 기능 및 로직

*   **수업 리포트 자동화**: 학생의 수업 내용, 지난 과제, 수업 일시 등을 자동으로 취합하여 알림톡으로 전송합니다.
*   **납부 알림 로직**: `수업회차`와 `수업구분`의 숫자를 비교하여, 특정 회차 도달 시 자동으로 수업료 납부 안내를 수행합니다.
*   **다중 연락처 지원**: 학부모 연락처(모/부)를 모두 확인하여 등록된 번호로 각각 메시지를 발송합니다.

## 🔧 유지보수 가이드

*   **메시지 양식 수정**: `HTTP Request - 수업 리포트` 노드 및 `HTTP Request - 수업료 납부 알림` 노드의 **Body** 설정을 수정하세요.
*   **납부 알림 조건 변경**: `If - 납부 알림 조건 체크` 노드의 로직을 수정하여 알림 발송 기준을 변경할 수 있습니다.
*   **데이터 필드 변경**: 노션 데이터베이스의 속성명이 변경될 경우, `Code` 노드 내의 데이터 매핑 부분을 업데이트해야 합니다.

# Daily Report 260120 워크플로우

본 워크플로우는 **Math4U** 시스템의 일환으로, 학생의 수업 데이터를 노션(Notion)에서 조회하여 수업 리포트를 생성하고 학부모에게 자동으로 발송하는 자동화 프로세스입니다.

## 🔄 워크플로우 개요
웹훅(Webhook)을 통해 특정 수업 데이터의 변경 사항을 감지하고, 관련 학생 및 수업 정보를 취합하여 수업 리포트 메시지를 구성합니다. 이후 수업료 납부 여부 등을 체크하여 필요한 알림을 발송합니다.

## 🔄 전체 흐름
1. **Webhook**: 외부(노션 등)로부터 수업 데이터 업데이트 신호를 수신합니다.
2. **Notion - Student_Study 재조회**: 업데이트된 수업 페이지의 상세 정보를 가져옵니다.
3. **Notion - Students 조회**: 해당 수업에 연결된 학생 정보를 조회합니다.
4. **Notion - Class_Study 조회**: 수업 과정 정보를 조회합니다.
5. **If - 지난수업 있는지 체크**: 이전 수업 데이터 존재 여부를 확인합니다.
6. **Notion - 지난수업 조회**: 이전 수업 데이터가 있을 경우 해당 정보를 가져옵니다.
7. **Code**: 수집된 데이터를 바탕으로 수업 리포트 메시지 본문과 수신자 정보를 가공합니다.
8. **HTTP Request - 수업 리포트**: 가공된 데이터를 바탕으로 학부모에게 수업 리포트 알림톡/메시지를 발송합니다.
9. **If - 납부 알림 조건 체크**: 수업료 납부 대상 여부를 판단합니다.
10. **HTTP Request - 수업료 납부 알림**: 미납 시 납부 안내 메시지를 발송합니다.
11. **Notion - 발송 완료 체크**: 발송 상태를 노션 데이터베이스에 업데이트합니다.

## ⚙️ 주요 기능
* **자동 리포트 생성**: 수업 내용, 지난 과제, 다음 수업 일정 등을 포함한 맞춤형 리포트 자동 생성.
* **다중 연락처 지원**: 학생의 어머니와 아버지 연락처를 모두 확인하여 메시지 발송.
* **상태 동기화**: 발송 완료 후 노션 내 상태 값을 업데이트하여 중복 발송 방지.

## 🔧 수정 및 유지보수
* **수업 리포트 양식**: `HTTP Request - 수업 리포트` 노드의 Body 항목을 수정하세요.
* **납부 알림 조건**: `If - 납부 알림 조건 체크` 노드에서 로직을 변경할 수 있습니다.
* **알림 메시지 내용**: `HTTP Request - 수업료 납부 알림` 노드의 Body 항목을 수정하세요.

* **Credential**: 
    * Notion API 토큰이 설정되어 있어야 합니다.

* **데이터 구조**: 노션 데이터베이스의 속성명(`이름`, `수업일시`, `수업회차` 등)이 변경될 경우 `Code` 노드의 데이터 매핑 코드를 반드시 수정해야 합니다.
* **Webhook**: 웹훅 URL이 외부에서 접근 가능한 상태인지 확인하십시오.

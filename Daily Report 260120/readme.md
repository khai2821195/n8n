## **📋 1. 시스템 개요**

이 워크플로우는 노션에서 수업 보고서가 작성되면 **카카오 알림톡**을 자동으로 발송하고, 완료 여부를 노션에 다시 기록합니다.

• **트리거**: 노션 자동화(Webhook) - 수업 보고서 작성 시 발생
• **발송 채널**: NHN Cloud 알림톡 (카카오톡)
• **처리 로직**:
    1. **수업 리포트**: 모든 학생에게 기본 발송
    2. **납부 안내**: `수업회차`가 `수업구분` 숫자와 일치할 때만 추가 발송

## **🛠 2. 노드별 상세 설정 (유지보수용)**

**🧠 Code 노드 (데이터 가공)**

노션의 복잡한 데이터를 알림톡 규격에 맞게 변환하는 핵심 로직입니다.[!IMPORTANT]
핵심 함수: extractNumber
노션 수식 필드가 숫자(4)든 문자("4회차")든 상관없이 숫자만 뽑아내어 비교 에러를 방지합니다.

• **변수 매핑**: 템플릿의 `#{이름}` 등은 코드 내 `paymentParams`의 키값과 대조됩니다.
• **기타안내**: 데이터가 없어도 빈 문자열(`""`)을 전송하여 발송 실패를 방지합니다.

### **🔀 If 노드 (분기 조건)**

• **조건**: `sendPaymentNotice` (Boolean)
• **True**: 수업료 납부 대상 (알림톡 발송 -> 노션 체크)
• **False**: 일반 수업 (바로 노션 체크)
****

### **✉️ HTTP Request (알림톡 발송)**

• **수업 리포트**: `{{ $json.reportData }}` 참조
• **납부 안내**: `{{ $('Code').item.json.paymentData }}` 참조 (**Code 노드 직접 지목**)
****

### **📝 Notion 노드 (체크박스 업데이트)**

• **동작**: `보고서 발송` 속성을 `true`로 변경
• **Page ID**: `{{ $('Webhook').item.json.body.data.id }}` (어떤 경로에서도 웹훅 ID 참조)
****

## **🚨 3. 문제 해결 (Troubleshooting)현상원인해결 방법**

**알림톡 발송 실패 (-3004)**

템플릿 변수명 불일치코드의 `paymentParams`와 NHN 콘솔의 `#{변수명}` 일치 확인

**납부 알림이 안 가요** 조건 불일치

노션의 `수업회차(S)`와 `수업구분` 숫자가 같은지 확인

**메시지 순서가 거꾸로 와요** 비동기 처리 특성

두 발송 노드 사이에 `Wait` 노드 (1~2초) 추가
****

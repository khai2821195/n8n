# Class → Student 자동생성 워크플로우

## ⚙️ 워크플로우 개요
본 워크플로우는 **Math4U** 시스템의 핵심 자동화 프로세스로, 노션(Notion)의 'Class_Study' 데이터베이스에 수업이 생성되면, 해당 수업에 배정된 학생들을 개별 'Student_Study' 페이지로 자동 생성하고 연결하는 역할을 수행합니다.

## 🔄 전체 흐름
1. **Webhook**: 노션 또는 외부 시스템으로부터 수업 생성 이벤트(POST 요청)를 수신합니다.
2. **Code - 입력 파싱**: 수신된 데이터에서 `classStudyId`와 `수업일정` 날짜 정보를 추출합니다.
3. **Class_Study Get**: 추출된 ID를 바탕으로 노션에서 해당 수업의 상세 정보를 조회합니다.
4. **Split Out - Student ID Split**: 수업에 연결된 학생 리스트(`property_students`)를 개별 항목으로 분리합니다.
5. **Student ID (Notion 조회)**: 분리된 각 학생의 상세 정보를 노션에서 조회합니다.
6. **Edit Fields**: 생성될 'Student_Study' 페이지의 제목(학생명 | 수업명)을 조합합니다.
7. **Create Page Student_Study**: 학생별 수업 페이지를 생성하고 수업일시, 학생, 수업, 수업 상세 정보를 관계형으로 연결합니다.
8. **Notion - 배정확인**: 모든 학생의 페이지 생성이 완료되면, 원본 'Class_Study' 페이지의 '배정확인' 체크박스를 활성화합니다.

## 🔧 수정 및 유지보수
- **입력 파라미터 변경**: `Code - 입력 파싱` 노드의 자바스크립트 코드를 수정하여 수신 데이터 구조를 변경할 수 있습니다.
- **생성 필드 수정**: `Create Page Student_Study` 노드에서 노션 데이터베이스에 저장될 속성(Properties)을 추가하거나 수정할 수 있습니다.
- **배정확인 로직**: `Notion - 배정확인` 노드에서 체크박스 외에 다른 상태값 업데이트가 필요할 경우 수정하십시오.

## 📋 환경 설정 및 크레덴셜
- **Notion Credential**: 노션 API 접근을 위한 토큰이 설정되어 있어야 합니다.
- **데이터베이스 ID**: 워크플로우 내 각 노드에 설정된 `databaseId` 및 `pageId`가 실제 사용 중인 노션 페이지와 일치하는지 확인하십시오.

## ⚠️ 주의사항
- **데이터 구조**: 노션 데이터베이스의 속성명(`수업일정`, `Students`, `Class` 등)이 변경될 경우, 각 노드의 매핑 설정을 반드시 업데이트해야 합니다.
- **API 제한**: 학생 수가 많을 경우 노션 API 호출 제한(Rate Limit)에 걸릴 수 있으므로, 대량의 데이터를 처리할 때는 n8n의 실행 속도 조절(Wait 노드 추가 등)을 고려하십시오.

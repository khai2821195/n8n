# 🔧 WF Status Updater (공통 서브워크플로우)

이 워크플로우는 n8n 내의 다양한 워크플로우 실행 상태를 중앙에서 관리하고, 노션(Notion) 페이지 업데이트 및 에러 발생 시 Slack 알림을 전송하기 위한 **공통 서브워크플로우**입니다.

## 📋 목적
- 워크플로우 실행 결과를 노션 데이터베이스에 기록하여 모니터링 효율화
- 에러 발생 시 즉각적인 Slack 알림을 통해 장애 대응 속도 향상
- 인프라 관련 일시적 오류(502, 503 등)를 필터링하여 불필요한 알림 방지

## ⚙️ 입력 파라미터
이 워크플로우를 호출할 때 다음 데이터를 전달해야 합니다.

| 파라미터명 | 필수 여부 | 설명 |
| :--- | :---: | :--- |
| `workflowId` | ✅ 필수 | 호출한 워크플로우의 고유 ID |
| `notionPageId` | ✅ 필수 | 상태를 업데이트할 노션 페이지 ID |
| `status` | 선택 | 상태값 (`✅ 정상` 또는 `❌ 에러`, 기본값: `✅ 정상`) |
| `errorMessage` | 선택 | 에러 발생 시 상세 메시지 |
| `workflowName` | 선택 | 워크플로우 이름 |

## 🚀 워크플로우 흐름
1. **워크플로우 호출 수신**: 상위 워크플로우로부터 데이터를 전달받습니다.
2. **입력값 검증**: 필수 파라미터 확인 및 한국 표준시(KST) 기준의 실행 시간(`lastRun`)을 생성합니다.
3. **노션 상태 업데이트**: Notion API(PATCH)를 사용하여 페이지의 `Status`, `Last Run`, `Error Message` 속성을 업데이트합니다.
4. **에러 여부 확인**: 상태가 `❌ 에러`인지 확인합니다.
5. **인프라 오류 제외**: 에러 메시지에 502, 503 등 인프라 관련 키워드가 포함된 경우 알림을 필터링합니다.
6. **Slack 알림**: 필터링을 통과한 에러에 대해 Slack 채널로 상세 내용을 전송합니다.
7. **결과 반환**: 최종 처리 결과를 호출 측으로 반환합니다.

## 🔑 필요 크레덴셜
- **Notion API**: 페이지 업데이트 권한이 있는 API 토큰
- **Slack**: 알림 메시지 발송 권한이 있는 Slack 토큰

## ⚠️ 주의사항
- **Notion 속성명**: 노션 데이터베이스에 `Status`(Select), `Last Run`(Date), `Error Message`(Rich Text) 속성이 반드시 존재해야 합니다.
- **Slack 채널**: 현재 `C0AM1MRP3BK` 채널로 고정되어 있으므로, 필요 시 노드 설정에서 채널 ID를 변경하십시오.
- **인프라 오류 필터링**: `If - 인프라 오류 제외` 노드에서 정규식(Regex)을 통해 특정 에러 메시지를 필터링하고 있습니다. 필요에 따라 제외할 에러 패턴을 추가할 수 있습니다.

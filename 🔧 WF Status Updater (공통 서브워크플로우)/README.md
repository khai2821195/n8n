# 🔧 WF Status Updater (공통 서브워크플로우)

이 워크플로우는 n8n 내의 다양한 워크플로우 실행 상태를 중앙에서 관리하고, 노션(Notion) 데이터베이스에 기록하며, 에러 발생 시 슬랙(Slack)으로 알림을 보내는 **공통 서브워크플로우**입니다.

## 📋 목적
- 워크플로우 실행 상태(성공/실패)를 노션 페이지에 실시간으로 업데이트합니다.
- 에러 발생 시 인프라 관련 오류(502, 503 등)를 필터링하여 불필요한 알림을 방지하고, 실제 비즈니스 로직 에러에 대해서만 슬랙 알림을 발송합니다.

## ⚙️ 입력 파라미터
이 워크플로우를 호출할 때 다음 데이터를 전달해야 합니다.

| 파라미터 | 필수 여부 | 설명 |
| :--- | :---: | :--- |
| `workflowId` | ✅ | 워크플로우 고유 ID |
| `notionPageId` | ✅ | 상태를 업데이트할 노션 페이지 ID |
| `status` | - | 상태값 (`✅ 정상` 또는 `❌ 에러`, 기본값: `✅ 정상`) |
| `workflowName` | - | 워크플로우 이름 |
| `errorMessage` | - | 에러 발생 시 상세 메시지 |

## 🔄 워크플로우 흐름
1. **워크플로우 호출 수신**: 상위 워크플로우로부터 데이터를 전달받습니다.
2. **입력값 검증**: 필수 파라미터(`workflowId`, `notionPageId`)를 확인하고 기본값을 설정합니다.
3. **노션 상태 업데이트**: Notion API(PATCH)를 통해 해당 페이지의 'Status', 'Last Run', 'Error Message' 속성을 업데이트합니다.
4. **에러 여부 확인**: 상태가 `❌ 에러`인지 확인합니다.
5. **If - 인프라 오류 제외**: 에러 메시지에 인프라 관련 키워드(502, 503, Bad gateway 등)가 포함되어 있는지 확인하여 알림 여부를 결정합니다.
6. **Slack 에러 알림**: 인프라 오류가 아닌 경우, 지정된 슬랙 채널로 에러 상세 내용과 워크플로우 링크를 전송합니다.
7. **결과 반환**: 호출한 워크플로우에 처리 결과를 반환합니다.

## 🔑 필요 크레덴셜
- **Notion API**: 노션 페이지 업데이트 권한이 있는 API 토큰이 필요합니다.
- **Slack**: 알림 메시지 발송을 위한 슬랙 앱 토큰 또는 웹훅 설정이 필요합니다.

## ⚠️ 주의사항
- **Notion 버전**: Notion API 버전은 `2022-06-28`을 기준으로 작성되었습니다.
- **슬랙 채널**: 현재 `C0AM1MRP3BK` 채널로 고정되어 있습니다. 환경에 따라 노드 설정을 변경하십시오.
- **인프라 오류 필터링**: `If - 인프라 오류 제외` 노드에서 정규표현식을 사용하여 필터링하므로, 추가적인 제외 키워드가 필요할 경우 해당 노드의 정규식 설정을 수정하십시오.

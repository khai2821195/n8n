# 🔧 WF Status Updater (공통 서브워크플로우)

이 워크플로우는 n8n 내의 다른 워크플로우들이 자신의 실행 상태(성공/실패)를 노션(Notion) 데이터베이스에 기록하고, 에러 발생 시 Slack으로 알림을 보내기 위해 설계된 **공통 서브워크플로우**입니다.

## 📋 목적
- 중앙화된 워크플로우 상태 관리 (노션 연동)
- 에러 발생 시 즉각적인 Slack 알림 및 모니터링
- 인프라 관련 일시적 오류(502, 503 등) 필터링을 통한 알림 피로도 감소

## ⚙️ 동작 방식
1. **워크플로우 호출 수신**: 상위 워크플로우로부터 실행 데이터(`workflowId`, `status`, `notionPageId` 등)를 전달받습니다.
2. **입력값 검증**: 필수 파라미터(`workflowId`, `notionPageId`)를 확인하고 기본값을 설정합니다.
3. **노션 상태 업데이트**: Notion API를 호출하여 해당 페이지의 'Status', 'Last Run', 'Error Message' 속성을 업데이트합니다.
4. **에러 여부 확인**: 상태가 `❌ 에러`인 경우에만 다음 단계로 진행합니다.
5. **인프라 오류 필터링**: 에러 메시지에 502, 503, 504 등 인프라 관련 키워드가 포함된 경우 알림을 제외합니다.
6. **Slack 알림**: 필터링을 통과한 에러에 대해 Slack 채널로 상세 내용을 전송합니다.
7. **결과 반환**: 호출한 워크플로우에 처리 결과를 반환합니다.

## 📥 입력 파라미터
| 파라미터 | 필수 여부 | 설명 |
| :--- | :---: | :--- |
| `workflowId` | ✅ | 호출한 워크플로우의 고유 ID |
| `notionPageId` | ✅ | 상태를 기록할 노션 페이지 ID |
| `status` | - | 상태 값 (`✅ 정상` 또는 `❌ 에러`, 기본값: `✅ 정상`) |
| `errorMessage` | - | 에러 발생 시 상세 메시지 |
| `workflowName` | - | 워크플로우 이름 |

## 🔑 필요 크레덴셜
이 워크플로우를 정상적으로 실행하려면 다음 크레덴셜이 n8n에 설정되어 있어야 합니다.
- **Notion API**: 데이터베이스 페이지 업데이트 권한이 있는 API 토큰
- **Slack**: 알림 메시지 전송 권한이 있는 Slack 앱 토큰

## ⚠️ 주의사항
- **Notion 속성명**: 노션 데이터베이스에 `Status`(Select), `Last Run`(Date), `Error Message`(Rich Text) 속성이 정확히 생성되어 있어야 합니다.
- **Slack 채널**: 현재 `C0AM1MRP3BK` 채널 ID로 고정되어 있습니다. 필요에 따라 노드 설정을 변경하십시오.
- **인프라 오류 필터링**: `If - 인프라 오류 제외` 노드에서 정규식(Regex)을 통해 특정 에러 코드를 필터링하고 있으므로, 필요 시 추가적인 에러 패턴을 정규식에 등록하여 관리할 수 있습니다.

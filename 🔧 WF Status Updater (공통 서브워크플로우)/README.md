# 🔧 WF Status Updater (공통 서브워크플로우)

이 워크플로우는 n8n 내의 다양한 워크플로우 실행 상태를 중앙에서 관리하고, 노션(Notion) 페이지에 기록하며, 에러 발생 시 Slack으로 알림을 보내는 **공통 서브워크플로우**입니다.

## 📋 목적
- 워크플로우 실행 상태(성공/실패)를 노션 데이터베이스에 실시간으로 동기화합니다.
- 에러 발생 시 특정 조건(인프라 오류 제외)에 따라 Slack으로 즉각적인 알림을 발송하여 운영 효율성을 높입니다.

## ⚙️ 동작 방식
1. **워크플로우 호출 수신**: 상위 워크플로우로부터 실행 데이터(ID, 상태, 에러 메시지 등)를 전달받습니다.
2. **입력값 검증**: 필수 파라미터(`workflowId`, `notionPageId`)를 확인하고 데이터를 정제합니다.
3. **노션 상태 업데이트**: Notion API를 통해 해당 페이지의 `Status`, `Last Run`, `Error Message` 속성을 업데이트합니다.
4. **에러 여부 확인**: 상태가 `❌ 에러`인 경우에만 다음 단계로 진행합니다.
5. **인프라 오류 필터링**: 502, 503 등 일시적인 인프라 오류는 알림에서 제외합니다.
6. **Slack 알림**: 필터링을 통과한 에러에 대해 Slack 채널로 상세 내용을 전송합니다.
7. **결과 반환**: 호출한 워크플로우에 처리 결과를 반환합니다.

## 📥 입력 파라미터
호출 시 다음 JSON 데이터를 전달해야 합니다.

| 파라미터 | 필수 여부 | 설명 |
| :--- | :---: | :--- |
| `workflowId` | ✅ | 워크플로우 고유 ID |
| `notionPageId` | ✅ | 업데이트할 노션 페이지 ID |
| `status` | - | 상태값 (`✅ 정상` 또는 `❌ 에러`) |
| `errorMessage` | - | 에러 발생 시 상세 메시지 |
| `workflowName` | - | 워크플로우 이름 |

## 🔑 필요 크레덴셜
- **Notion API**: 페이지 업데이트 권한이 있는 API 토큰
- **Slack**: 메시지 전송 권한이 있는 API 토큰

## ⚠️ 주의사항
- **Notion 버전**: Notion API 버전은 `2022-06-28`을 기준으로 작성되었습니다.
- **인프라 오류 필터링**: `If - 인프라 오류 제외` 노드에서 정규식(Regex)을 통해 특정 에러 코드(502, 503 등)를 필터링하고 있습니다. 필요에 따라 정규식 패턴을 수정하여 관리 범위를 조정할 수 있습니다.
- **Slack 채널**: 현재 특정 채널 ID(`C0AM1MRP3BK`)로 고정되어 있으므로, 환경에 맞게 노드 설정을 확인하십시오.

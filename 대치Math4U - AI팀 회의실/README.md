# 대치Math4U - AI팀 회의실

이 워크플로우는 대치Math4U의 주요 의사결정 과정을 돕기 위해 설계된 **AI 가상 회의 시스템**입니다. 사용자가 회의 아젠다를 입력하면, 4명의 가상 전문가(닐, 존, 빌, 팀)가 각자의 전문 분야 관점에서 의견을 제시하여 전략 수립을 지원합니다.

---

## 📋 워크플로우 개요
- **이름**: 대치Math4U - AI팀 회의실
- **목적**: 특정 아젠다에 대해 다각적인 전문가 관점의 피드백을 즉시 확보
- **핵심 기술**: n8n Form Trigger, Google Gemini (LangChain), 데이터 파싱

## ⚙️ 동작 방식 (Node Flow)
1. **회의 시작 폼 (Form Trigger)**: 사용자가 회의 주제(아젠다)를 입력합니다.
2. **입력 파싱 (Set Node)**: 입력된 아젠다와 현재 날짜를 시스템 변수로 정규화합니다.
3. **AI 전문가 그룹 (Google Gemini)**: 4개의 독립적인 Gemini 노드가 각기 다른 페르소나를 부여받아 의견을 생성합니다.
    - **닐 (Neil Patel)**: 마케팅/SEO/SNS 전문가
    - **존 (Jony Ive)**: 디자인/브랜딩/사용자 경험 전문가
    - **빌 (Bill Gates)**: 운영 자동화/시스템 효율 전문가
    - **팀 (Tim Cook)**: 학생 관리/학부모 상담/고객 만족 전문가
4. **회의록 생성 (Set Node)**: 각 전문가의 의견을 취합하여 마크다운 형식의 회의록을 작성합니다.

## 🔑 필수 설정 및 크레덴셜
- **Google Gemini API**: 각 AI 노드에서 Gemini 모델을 사용하기 위해 유효한 Google Gemini API Key가 설정된 크레덴셜이 필요합니다.
- **n8n 환경**: `n8n.math4u.co.kr` 서버 환경에서 최적화되어 있습니다.

## 💡 활용 방법
1. 워크플로우를 활성화(`Active`)합니다.
2. 제공된 폼 URL에 접속하여 논의하고자 하는 주제를 입력합니다.
3. 결과물로 출력된 마크다운 회의록을 복사합니다.
4. **지니(Claude.ai)**에게 해당 내용을 전달하여 최종 액션 아이템을 도출합니다.

## ⚠️ 주의사항
- **토큰 제한**: 각 AI 노드는 `maxOutputTokens`가 500으로 설정되어 있습니다. 답변이 잘릴 경우 해당 노드의 설정을 조정하십시오.
- **모델 버전**: 현재 `gemini-3.1-flash-lite-preview` 모델을 사용 중입니다. API 가용성에 따라 모델을 변경할 수 있습니다.
- **데이터 보안**: 민감한 학생 정보나 내부 기밀 사항을 입력할 때는 주의를 요합니다.

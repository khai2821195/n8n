# 대치Math4U - AI팀 회의실

이 워크플로우는 대치Math4U의 주요 의사결정 및 전략 수립을 위해 4명의 가상 전문가(닐, 존, 빌, 팀)가 각자의 관점에서 의견을 제시하는 **AI 기반 브레인스토밍 자동화 도구**입니다.

## 📋 워크플로우 개요
사용자가 회의 아젠다를 입력하면, n8n과 Google Gemini 모델을 활용하여 4명의 페르소나가 다각도로 분석한 회의록을 자동으로 생성합니다.

## 🚀 동작 방식
1. **회의 시작 폼 (Form Trigger):** 웹 폼을 통해 논의할 아젠다를 입력받습니다.
2. **입력 파싱 (Set Node):** 입력된 아젠다와 현재 날짜를 추출하여 워크플로우 전반에서 사용할 변수로 설정합니다.
3. **AI 전문가 그룹 (Google Gemini):** 각 전문가별 페르소나를 부여받은 4개의 Gemini 노드가 병렬로 의견을 생성합니다.
    * **닐 (Neil Patel):** 마케팅 및 SEO 관점의 전략 제안
    * **존 (Jony Ive):** 디자인 및 브랜드 일관성 관점의 제안
    * **빌 (Bill Gates):** 운영 자동화 및 시스템 효율성 관점의 제안
    * **팀 (Tim Cook):** 학생 관리 및 학부모 신뢰 관점의 제안
4. **회의록 작성 (Set Node):** 각 전문가의 의견을 취합하여 마크다운 형식의 회의록을 생성합니다.

## 🛠 필수 설정 및 요구사항
* **n8n 버전:** 1.0 이상 권장
* **크레덴셜:** 
    * `Google Gemini API Key`가 설정되어 있어야 합니다.
* **모델:** `gemini-3.1-flash-lite-preview` 모델을 사용하므로, 해당 모델 접근 권한이 있는 API 키가 필요합니다.

## 💡 활용 팁
* 생성된 회의록 하단의 '지니 검토 필요' 섹션을 활용하여, 최종적으로 Claude.ai 등 LLM에 내용을 복사/붙여넣기 하면 더욱 정교한 액션 아이템 도출이 가능합니다.
* 특정 페르소나의 말투나 역할이 변경되어야 할 경우, 각 Gemini 노드의 `System Prompt`를 수정하여 커스터마이징할 수 있습니다.

## ⚠️ 주의사항
* 본 워크플로우는 AI가 생성한 의견을 제공하므로, 최종 의사결정 전 반드시 담당자의 검토가 필요합니다.
* `maxOutputTokens`가 500으로 설정되어 있으므로, 너무 긴 아젠다 입력 시 답변이 잘릴 수 있습니다. 필요에 따라 토큰 값을 조정하십시오.

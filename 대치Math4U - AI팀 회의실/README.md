# 대치Math4U - AI팀 회의실

본 워크플로우는 대치Math4U의 주요 의사결정을 위해 4명의 가상 페르소나(닐, 존, 빌, 팀)가 각자의 전문 분야 관점에서 아젠다를 분석하고 의견을 제시하는 **AI 기반 브레인스토밍 자동화 도구**입니다.

## 🚀 워크플로우 목적
학원 운영 및 마케팅 관련 아젠다를 입력하면, 마케팅, 디자인, 운영 자동화, 학생 관리라는 4가지 핵심 관점에서 즉각적인 피드백을 생성하여 효율적인 의사결정을 지원합니다.

## ⚙️ 동작 방식
1. **회의 시작 폼 (Form Trigger)**: 사용자가 회의 주제(Agenda)를 입력합니다.
2. **입력 파싱 (Set Node)**: 입력된 아젠다와 현재 날짜를 워크플로우 전반에서 사용할 수 있도록 변수로 정리합니다.
3. **AI 페르소나 분석 (Google Gemini)**:
    * **닐 (Neil Patel)**: SEO 및 마케팅 관점의 콘텐츠 전략 제안
    * **존 (Jony Ive)**: 브랜드 일관성 및 디자인 미학 관점 제안
    * **빌 (Bill Gates)**: n8n 기반 운영 자동화 및 효율성 제안
    * **팀 (Tim Cook)**: 학생 관리 및 학부모 신뢰 중심의 상담 관점 제안
4. **회의록 생성 (Set Node)**: 각 페르소나의 의견을 취합하여 마크다운 형식의 회의록을 작성합니다.

## 🛠 필수 설정 및 요구사항
* **Google Gemini API Key**: 각 페르소나 노드에서 `gemini-3.1-flash-lite-preview` 모델을 사용하기 위해 n8n의 Google Gemini Credential 설정이 필요합니다.
* **모델 권한**: 사용 중인 API 키가 `gemini-3.1-flash-lite-preview` 모델에 접근 가능한지 확인하십시오.

## 📝 활용 가이드
1. 워크플로우를 활성화(`Active`)합니다.
2. n8n 폼 URL을 통해 접속하여 논의할 아젠다를 입력합니다.
3. 생성된 회의록 하단의 '지니 검토 필요' 섹션을 참고하여, 최종 액션 아이템을 확정합니다.

## ⚠️ 주의사항
* 본 워크플로우는 생성형 AI를 활용하므로, 답변의 구체적인 내용은 매번 달라질 수 있습니다.
* 생성된 결과물은 초안으로 활용하시고, 최종 의사결정 시 반드시 담당자의 검토를 거치시기 바랍니다.
* `maxOutputTokens`가 500으로 설정되어 있으므로, 매우 긴 아젠다 입력 시 답변이 잘릴 수 있습니다. 필요시 조정하십시오.

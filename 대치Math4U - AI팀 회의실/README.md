# 대치Math4U - AI팀 회의실

이 워크플로우는 대치Math4U의 주요 의사결정 과정을 돕기 위해 설계된 **AI 가상 회의 시스템**입니다. 하나의 아젠다를 입력하면 마케팅, 디자인, 운영 자동화, 학생 관리 분야의 전문가 페르소나를 가진 AI 모델들이 각자의 관점에서 전략적인 의견을 제시합니다.

## 🚀 워크플로우 목적
학원 운영 및 마케팅 전략 수립 시, 다양한 직무의 관점을 즉각적으로 반영하여 의사결정의 질을 높이고 아이디어를 구체화하기 위해 제작되었습니다.

## ⚙️ 동작 방식
1. **회의 시작 폼 (Trigger):** 사용자가 회의 주제(Agenda)를 입력합니다.
2. **입력 파싱 (Set):** 입력된 아젠다와 현재 날짜를 시스템 변수로 정리합니다.
3. **전문가 의견 수렴 (Gemini AI):** 4명의 페르소나(닐, 존, 빌, 팀)가 각각의 전문 분야 관점에서 의견을 생성합니다.
   - **닐 (마케터):** SEO, SNS 트렌드, 학부모 클릭률 중심
   - **존 (디자이너):** 브랜드 일관성, 모바일 가독성, 미학적 관점
   - **빌 (운영자동화):** 시스템 효율성, 자동화 가능 여부, 시간 절약
   - **팀 (학생관리):** 학생 만족도, 학부모 신뢰, 상담 효율성
4. **회의록 생성:** 각 전문가의 의견을 종합하여 마크다운 형식의 회의록을 출력합니다.

## 🛠 필수 설정 및 요구사항
- **n8n 버전:** 1.0 이상 권장
- **크레덴셜:** Google Gemini API Key가 설정되어 있어야 합니다.
- **모델:** `gemini-3.1-flash-lite-preview` 모델을 사용하므로, 해당 모델 접근 권한이 있는 API Key가 필요합니다.

## 📝 사용 방법
1. n8n 워크플로우를 활성화합니다.
2. 폼 트리거 URL을 통해 접속하거나 n8n 에디터에서 'Execute Workflow'를 실행합니다.
3. 입력창에 논의하고 싶은 주제를 입력합니다.
4. 출력된 회의록 내용을 확인하고, 하단 가이드에 따라 `Claude.ai(지니)`를 통해 최종 액션 아이템을 확정합니다.

## ⚠️ 주의사항
- 본 워크플로우는 AI가 생성한 의견을 제공하므로, 최종 결정 전 반드시 담당자의 검토가 필요합니다.
- `maxOutputTokens`가 500으로 설정되어 있으므로, 답변이 너무 길어질 경우 조정이 필요할 수 있습니다.
- 회의록 하단의 '지니 검토 필요' 섹션은 실제 업무 프로세스에 맞춰 활용하시기 바랍니다.

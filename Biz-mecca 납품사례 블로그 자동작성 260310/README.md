# Biz-mecca 납품사례 블로그 자동작성 워크플로우

본 워크플로우는 비즈메카(Biz-mecca) 납품 정보를 입력받아 네이버 블로그용 홍보 글 초안을 자동으로 생성하고, 노션(Notion)에 저장하는 자동화 시스템입니다.

---

## 📋 워크플로우 개요
- **목적**: 납품 사례를 기반으로 한 블로그 콘텐츠 제작 시간 단축 및 마케팅 효율화
- **주요 기능**: 
    - 사용자 입력 폼을 통한 데이터 수집
    - Gemini AI를 활용한 2단계(초안 및 심화) 블로그 글 작성
    - 노션 데이터베이스 자동 업데이트

---

## 🔄 전체 흐름
1. **납품사례 폼 입력 (Form Trigger)**: 제품명, 거래처, 납품처 유형 등 필수 정보 입력
2. **입력값 정리**: 입력된 데이터를 워크플로우 내부 변수로 매핑
3. **[비활성] 제품 URL 크롤링/파싱**: (옵션) 제품 상세 페이지 정보를 직접 가져올 때 사용
4. **Gemini 프롬프트 조립 및 호출**: 1차 블로그 초안 생성
5. **노션에 초안 저장**: 생성된 초안을 노션 DB에 기록
6. **심화 프롬프트 조립 및 2차 호출**: 블로그 글의 퀄리티를 높이기 위한 심화 내용 생성
7. **노션 업데이트**: 심화된 본문을 기존 노션 페이지에 업데이트 및 블록 추가

---

## 🔧 설정 및 요구사항

### 1. 필수 크레덴셜
- **Google Gemini API**: 블로그 글 생성을 위해 필요합니다.
- **Notion API**: 생성된 글을 저장할 데이터베이스 접근 권한이 필요합니다.

### 2. 환경 설정
- **노션 DB ID**: '노션에 초안 저장' 노드에서 본인의 노션 데이터베이스 ID를 정확히 입력해야 합니다.
- **프롬프트 수정**: 'Gemini 프롬프트 조립' 및 '심화 프롬프트 조립' 노드에서 브랜드 톤앤매너에 맞게 프롬프트를 수정하여 사용하세요.

### 3. URL 크롤링 활성화
- 현재 제품 URL 크롤링 기능은 비활성화되어 있습니다. 필요 시 '제품 URL 크롤링' 및 '제품 정보 파싱' 노드의 **Disabled** 설정을 해제하십시오.

## ⚠️ 주의사항
- **크롤링 주의**: 비즈메카 사이트의 구조 변경 시 크롤링 노드의 파싱 로직(Code 노드) 수정이 필요할 수 있습니다.
- **API 비용**: Gemini API 사용량에 따라 비용이 발생할 수 있으므로, 테스트 시에는 적절한 모델을 선택하여 사용하시기 바랍니다.
- **폼 입력**: 폼의 필수 항목이 누락되지 않도록 주의하십시오.

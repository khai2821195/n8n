# Biz-mecca 납품사례 블로그 자동작성 워크플로우

본 워크플로우는 비즈메카(Biz-mecca) 납품 정보를 입력받아, AI(Gemini)를 활용해 네이버 블로그용 홍보 글을 자동으로 생성하고 노션(Notion)에 저장하는 자동화 시스템입니다.

## 📋 워크플로우 개요
- **목적**: 납품사례 데이터를 기반으로 전문적인 블로그 포스팅 초안을 신속하게 작성하여 업무 효율성 증대
- **핵심 기능**: 
    - 사용자 입력 폼을 통한 데이터 수집
    - Gemini AI를 활용한 1차 초안 작성 및 2차 심화 콘텐츠 생성
    - 노션 데이터베이스 자동 연동 및 저장

## 🔄 전체 흐름
1. **납품사례 폼 입력 (Form Trigger)**: 제품명, 거래처, 수량 등 필수 정보 입력
2. **입력값 정리**: 수집된 데이터를 워크플로우 내부 변수로 매핑
3. **[비활성] 제품 정보 파싱**: (필요 시) 제품 URL을 크롤링하여 상세 정보를 자동으로 추출
4. **Gemini API 호출**: 
    - 1차: 기본 블로그 초안 생성
    - 2차: 심화 프롬프트를 통한 전문적인 포스팅 내용 보강
5. **노션 저장**: 생성된 초안 및 심화 본문을 노션 데이터베이스에 업데이트

## 🔧 설정 및 사용법

### 1. 필수 환경 설정
- **Gemini API Key**: AI 모델 호출을 위해 `n8n`의 Credentials 설정에 Gemini API Key가 등록되어 있어야 합니다.
- **Notion API Key & DB ID**: 노션 연동을 위해 `Notion API` 자격 증명과 데이터를 저장할 `Database ID`가 필요합니다.

### 2. 수정이 필요할 때
- **폼 항목 변경**: `납품사례 폼 입력` 노드에서 필드를 추가하거나 수정할 수 있습니다.
- **AI 프롬프트 수정**: `Gemini 프롬프트 조립` 및 `심화 프롬프트 조립` 노드에서 블로그의 톤앤매너를 변경할 수 있습니다.
- **데이터베이스 변경**: `노션에 초안 저장` 노드에서 타겟 데이터베이스 ID를 확인하세요.
- **크롤링 활성화**: 제품 URL을 통한 자동 정보 수집이 필요한 경우, 비활성화된 `제품 URL 크롤링` 및 `제품 정보 파싱` 노드를 활성화하십시오.

## ⚠️ 주의사항
- **크롤링 노드**: 현재 `제품 URL 크롤링` 노드는 비활성화 상태입니다. 웹사이트 구조 변경 시 파싱 로직(Code 노드)의 수정이 필요할 수 있습니다.
- **API 비용**: Gemini API 호출 횟수에 따라 비용이 발생할 수 있으므로, 대량 처리 시 API 사용량을 모니터링하십시오.
- **데이터 보안**: 입력 폼에 민감한 개인정보나 기밀 정보가 포함되지 않도록 주의하십시오.

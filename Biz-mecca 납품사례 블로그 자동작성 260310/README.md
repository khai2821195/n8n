# Biz-mecca 납품사례 블로그 자동작성 워크플로우

본 워크플로우는 비즈메카(Biz-mecca) 납품 정보를 입력받아, 네이버 블로그에 게시할 수 있는 홍보용 포스팅 초안을 자동으로 생성하고 노션(Notion)에 저장하는 자동화 시스템입니다.

## 📋 워크플로우 개요
- **목적**: 납품사례 데이터를 기반으로 AI(Gemini)를 활용한 블로그 원고 자동 생성
- **주요 기능**:
    - 사용자 입력 폼을 통한 납품 정보 수집
    - Gemini API를 활용한 1차 초안 생성 및 2차 심화본문 작성
    - 노션 데이터베이스에 결과물 자동 저장

## 🔄 전체 흐름
1. **납품사례 폼 입력**: 사용자로부터 제품명, 특징, 거래처, 수량, URL, 메모 등을 입력받습니다.
2. **입력값 정리**: 입력된 데이터를 워크플로우 내에서 사용하기 적합한 변수 형태로 가공합니다.
3. **[비활성] 제품 정보 크롤링/파싱**: 제품 URL을 통해 상세 정보를 가져옵니다. (현재 비활성 상태)
4. **Gemini 프롬프트 조립**: 수집된 정보를 바탕으로 블로그 포스팅을 위한 프롬프트를 구성합니다.
5. **Gemini API 호출**: AI를 통해 블로그 초안을 생성합니다.
6. **노션 저장**: 생성된 초안을 노션 데이터베이스에 저장합니다.
7. **심화 프롬프트 조립 및 호출**: 1차 초안을 바탕으로 내용을 보강하여 심화본문을 생성합니다.
8. **최종 업데이트**: 노션에 심화본문을 업데이트하고 완료 메시지를 출력합니다.

## ⚙️ 환경 설정 및 크레덴셜
이 워크플로우를 정상적으로 실행하기 위해 다음 항목이 필요합니다.

- **Gemini API Key**: Google AI Studio에서 발급받은 API 키가 필요합니다.
- **Notion API Token**: 노션 데이터베이스에 접근하기 위한 통합(Integration) 토큰이 필요합니다.
- **Notion Database ID**: 블로그 초안이 저장될 노션 페이지 또는 데이터베이스의 ID가 필요합니다.

## 🔧 수정 및 커스텀 가이드
- **폼 항목 변경**: `납품사례 폼 입력` 노드에서 필드를 추가하거나 수정할 수 있습니다.
- **프롬프트 수정**: `Gemini 프롬프트 조립` 또는 `심화 프롬프트 조립` 노드에서 블로그의 톤앤매너를 조정할 수 있습니다.
- **크롤링 활성화**: 제품 URL 기반의 자동 정보 수집이 필요한 경우, `제품 URL 크롤링` 및 `제품 정보 파싱` 노드의 **Disabled** 설정을 해제하십시오.
- **노션 연동**: `노션에 초안 저장` 노드에서 연동할 데이터베이스 ID를 본인의 환경에 맞게 설정하십시오.

## ⚠️ 주의사항
- **크롤링 기능**: 현재 제품 URL 크롤링 노드는 비활성화되어 있습니다. 활성화 시 대상 사이트의 `robots.txt` 정책을 준수하십시오.
- **API 비용**: Gemini API 호출 시 사용량에 따른 비용이 발생할 수 있습니다.
- **데이터 정합성**: AI가 생성한 결과물은 게시 전 반드시 사람이 검토하여 오타나 사실관계 오류를 확인하시기 바랍니다.

# Biz-mecca 납품사례 블로그 자동작성 워크플로우

본 워크플로우는 비즈메카(Biz-mecca) 납품 정보를 입력받아 네이버 블로그용 홍보 글을 자동으로 생성하고, 노션(Notion)에 저장하는 자동화 시스템입니다.

## 📋 워크플로우 개요
- **목적**: 납품 사례를 기반으로 전문적인 블로그 초안을 생성하여 업무 효율성 증대
- **핵심 기능**: 
    - 사용자 폼 입력을 통한 데이터 수집
    - Gemini AI를 활용한 2단계 블로그 글 작성 (초안 생성 및 심화 보완)
    - 노션 데이터베이스 자동 저장

## 🔄 전체 흐름
1. **납품사례 폼 입력 (Form Trigger)**: 제품명, 거래처, 수량 등 필수 정보를 입력받습니다.
2. **입력값 정리**: 입력된 데이터를 워크플로우 내에서 사용하기 쉽게 변수화합니다.
3. **[비활성] 제품 정보 크롤링/파싱**: 제품 URL을 통해 상세 정보를 추출합니다. (현재 비활성 상태)
4. **Gemini 프롬프트 조립 및 호출**: 수집된 정보를 바탕으로 1차 블로그 초안을 생성합니다.
5. **노션 초안 저장**: 생성된 초안을 노션 데이터베이스에 저장합니다.
6. **심화 프롬프트 조립 및 2차 호출**: 초안을 바탕으로 내용을 보강하고 전문성을 높인 심화본을 생성합니다.
7. **노션 업데이트**: 최종 심화본을 노션에 업데이트하고 완료 메시지를 출력합니다.

## 🔧 설정 및 요구사항
- **필수 크레덴셜**:
    - **Google Gemini API Key**: 텍스트 생성 및 분석을 위해 필요합니다.
    - **Notion API Key**: 블로그 초안 저장 및 업데이트를 위해 필요합니다.
- **노션 데이터베이스**:
    - 워크플로우 내 '노션에 초안 저장' 노드에서 대상 데이터베이스 ID를 설정해야 합니다.

## ⚠️ 주의사항 및 수정 가이드
- **크롤링 기능 활성화**: 제품 URL을 통한 자동 정보 수집이 필요한 경우, `제품 URL 크롤링` 및 `제품 정보 파싱` 노드의 비활성 설정을 해제(Enable)하십시오.
- **프롬프트 수정**: 블로그의 톤앤매너를 변경하고 싶다면 `Gemini 프롬프트 조립` 또는 `심화 프롬프트 조립` 노드의 내용을 수정하십시오.
- **폼 항목 변경**: `납품사례 폼 입력` 노드에서 수집 항목을 추가하거나 삭제하여 비즈니스 환경에 맞게 커스터마이징할 수 있습니다.

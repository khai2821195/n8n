# Class_Study → Student_Study 수업일정 동기화

본 워크플로우는 Notion의 `Class_Study` 데이터베이스에서 수업 일정이 변경되었을 때, 해당 수업에 연결된 학생들의 `Student_Study` 데이터베이스 내 수업 일시를 자동으로 동기화하는 n8n 자동화 프로세스입니다.

## 1. 워크플로우 목적
- `Class_Study` 데이터베이스의 수업 정보(날짜 및 시간)가 수정될 경우, 관계형(Relation)으로 연결된 모든 학생의 개별 학습 일정(`Student_Study`)을 최신 상태로 유지합니다.
- 수동으로 일정을 수정해야 하는 번거로움을 줄이고 데이터 정합성을 보장합니다.

## 2. 동작 방식 (Node Flow)

1.  **Class_Study 변경 감지 (Notion Trigger)**
    - `Class_Study` 데이터베이스의 페이지 업데이트 이벤트를 실시간으로 감지합니다.
2.  **수업일정 및 학생ID 추출 (Code Node)**
    - 업데이트된 페이지에서 '수업일정' 속성(시작일/종료일)을 추출합니다.
    - 해당 수업에 연결된 학생들의 Notion 페이지 ID 리스트를 가져옵니다.
    - 데이터가 누락되었거나 연결된 학생이 없는 경우 스킵 처리를 위한 플래그를 생성합니다.
3.  **업데이트 필요 여부 (If Node)**
    - 추출 단계에서 생성된 `skip` 플래그를 확인합니다.
    - `skip: false`인 경우에만 업데이트 로직을 수행합니다.
4.  **Student_Study 수업일시 업데이트 (Notion Node)**
    - 연결된 각 학생의 페이지 ID를 기반으로 `Student_Study` 데이터베이스의 '수업일시' 속성을 새로운 일정으로 업데이트합니다.
5.  **스킵 로그 (No-Op Node)**
    - 업데이트가 필요 없는 경우(날짜 정보 없음 또는 학생 미연결) 해당 흐름을 종료합니다.

## 3. 필수 설정 및 크레덴셜
- **Notion Credentials**: `Class_Study` 및 `Student_Study` 데이터베이스에 접근 가능한 API 토큰이 필요합니다.
- **데이터베이스 권한**: n8n 연동에 사용된 Notion 통합(Integration) 계정이 해당 데이터베이스에 대해 '읽기/쓰기' 권한을 가지고 있어야 합니다.
- **속성명 확인**: Notion 데이터베이스 내 속성 이름(`수업일정`, `수업일시`)이 워크플로우 코드와 일치하는지 확인하십시오.

## 4. 주의사항
- **관계형 속성**: `Class_Study` 데이터베이스에서 학생을 연결하는 관계형 속성이 반드시 설정되어 있어야 합니다.
- **날짜 형식**: Notion의 '날짜' 속성 타입을 사용해야 하며, 시간 포함(Include Time) 옵션이 활성화되어 있어야 정확한 동기화가 가능합니다.
- **대량 업데이트**: 한 번에 많은 학생이 연결된 수업을 수정할 경우, Notion API 호출 제한(Rate Limit)이 발생할 수 있으므로 주의가 필요합니다.

# Class_Study → Student_Study 수업일정 동기화

본 워크플로우는 Notion의 `Class_Study` 데이터베이스에서 수업 일정이 변경될 때, 해당 수업에 연결된 학생들의 `Student_Study` 데이터베이스 내 수업 일시를 자동으로 동기화하는 n8n 자동화 프로세스입니다.

## 1. 워크플로우 목적
- `Class_Study` 데이터베이스의 '수업일정' 속성 변경 시, 관계형(Relation)으로 연결된 모든 학생의 수업 일시를 최신 상태로 유지합니다.
- 수동으로 여러 데이터베이스를 업데이트해야 하는 번거로움을 제거하여 데이터 정합성을 확보합니다.

## 2. 동작 방식 (Node Flow)
1. **Class_Study 변경 감지 (Notion Trigger)**: `Class_Study` 데이터베이스에서 페이지 업데이트 이벤트가 발생하면 워크플로우가 실행됩니다.
2. **수업일정 및 학생ID 추출 (Code Node)**: 
   - 업데이트된 페이지에서 '수업일정' 속성(시작/종료 시간)을 추출합니다.
   - 관계형 속성에서 연결된 학생들의 고유 ID(studentPageId)를 추출합니다.
   - 필수 데이터(날짜 또는 학생ID)가 없는 경우 스킵 처리합니다.
3. **업데이트 필요 여부 (If Node)**: 추출 과정에서 유효한 데이터가 확인된 경우에만 다음 단계로 진행합니다.
4. **Student_Study 수업일시 업데이트 (Notion Node)**: 추출된 학생 ID를 기반으로 `Student_Study` 데이터베이스의 '수업일시' 속성을 업데이트합니다.
5. **스킵 로그 (NoOp Node)**: 데이터가 부족하여 업데이트가 불필요한 경우 해당 흐름을 종료합니다.

## 3. 필수 설정 (Prerequisites)
- **Notion Credentials**: Notion API 접근 권한이 설정된 자격 증명이 필요합니다.
- **데이터베이스 권한**: 워크플로우에서 사용하는 `Class_Study` 및 `Student_Study` 데이터베이스에 n8n 연동(Integration) 권한이 부여되어 있어야 합니다.
- **속성 이름 확인**: Notion 데이터베이스 내 속성 이름(`수업일정`, `수업일시`)이 워크플로우 코드와 일치하는지 확인하십시오.

## 4. 주의사항
- **관계형 속성**: `Class_Study` 데이터베이스에서 학생과 연결된 관계형 속성이 반드시 존재해야 합니다.
- **데이터 형식**: '수업일정' 속성은 Notion의 '날짜(Date)' 타입이어야 하며, 시간 포함 설정이 되어 있어야 정상적으로 동기화됩니다.
- **대량 업데이트**: 한 번에 너무 많은 학생이 연결된 수업을 수정할 경우, Notion API 호출 제한(Rate Limit)에 걸릴 수 있으므로 주의가 필요합니다.

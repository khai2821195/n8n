# Class_Study → Student_Study 수업일정 동기화

본 워크플로우는 Notion의 `Class_Study` 데이터베이스에서 수업 일정이 변경될 때, 해당 수업에 연결된 학생들의 `Student_Study` 데이터베이스 정보를 자동으로 업데이트하여 일정을 동기화하는 n8n 자동화 프로세스입니다.

## 📋 워크플로우 목적
수업 관리 데이터베이스(`Class_Study`)의 수업 시간 변경 사항을 실시간으로 감지하여, 개별 학생의 학습 일정 데이터베이스(`Student_Study`)에 즉시 반영함으로써 수동 업데이트의 번거로움을 줄이고 데이터 정합성을 유지합니다.

## ⚙️ 워크플로우 동작 방식

1.  **Class_Study 변경 감지 (Notion Trigger)**
    *   `Class_Study` 데이터베이스의 페이지 업데이트 이벤트를 실시간으로 감지합니다.
2.  **수업일정 및 학생ID 추출 (Code Node)**
    *   변경된 페이지에서 '수업일정' 속성(시작/종료 시간)을 추출합니다.
    *   해당 수업에 연결된 관계형(Relation) 속성에서 학생들의 페이지 ID 목록을 가져옵니다.
    *   데이터가 없거나 필수 정보가 누락된 경우, 이후 프로세스를 건너뛰도록 처리합니다.
3.  **업데이트 필요 여부 (If Node)**
    *   추출된 데이터에 오류가 없는지 확인합니다.
    *   `skip` 값이 `false`인 경우에만 업데이트 프로세스를 진행합니다.
4.  **Student_Study 수업일시 업데이트 (Notion Node)**
    *   추출된 각 학생의 페이지 ID를 기반으로 `Student_Study` 데이터베이스의 '수업일시' 속성을 최신 수업 일정으로 업데이트합니다.
5.  **스킵 로그 (NoOp Node)**
    *   업데이트가 필요 없는 경우(데이터 누락 등) 해당 흐름을 종료하고 로그를 남깁니다.

## 🔑 필수 설정 (Credentials & Environment)

*   **Notion Credentials**: 
    *   `Class_Study` 및 `Student_Study` 데이터베이스에 접근 가능한 Notion API 토큰이 필요합니다.
    *   해당 토큰은 두 데이터베이스에 대한 읽기/쓰기 권한을 모두 가지고 있어야 합니다.
*   **Database IDs**:
    *   노드 설정 내 `databaseId` 및 `pageId`가 실제 사용 중인 Notion 데이터베이스와 올바르게 매핑되어 있는지 확인하십시오.

## ⚠️ 주의사항

*   **속성 이름**: Notion 데이터베이스의 속성 이름(`수업일정`, `수업일시`)이 코드 노드 및 업데이트 노드에 명시된 이름과 정확히 일치해야 합니다. 변경 시 코드 노드의 스크립트 수정이 필요할 수 있습니다.
*   **관계형 속성**: `Class_Study` 내에서 학생이 연결된 관계형 속성이 하나 이상 존재해야 정상적으로 작동합니다.
*   **데이터 타입**: '수업일정' 속성은 Notion의 'Date' 타입이어야 하며, 시간 포함 여부 설정에 따라 업데이트 결과가 달라질 수 있습니다.

## 🛠 기술 스택
*   **n8n**: 워크플로우 자동화 엔진
*   **Notion API**: 데이터베이스 연동 및 데이터 동기화

# 등록일원부 자동생성 - 수업종료 워크플로우

본 워크플로우는 Notion의 `Student_Study` 데이터베이스에서 수업이 종료되었을 때, 이를 감지하여 자동으로 '등록일원부' 레코드를 생성하는 자동화 시스템입니다. 수업료 정보와 학생 정보를 통합하여 납부 회차를 계산하고, 최종적으로 등록일원부 DB에 데이터를 기록합니다.

## 🔄 전체 흐름

1.  **Notion Trigger**: `Student_Study` 데이터베이스의 변경 사항을 감지합니다.
2.  **Code (종료 체크)**: '수업완료' 체크박스 확인 및 수업 회차와 수업 구분 데이터를 비교하여 종료 여부를 판단합니다.
3.  **Notion (Class 조회)**: 해당 수업의 수업료 정보를 가져옵니다.
4.  **Code (수업료 추출)**: 조회된 Class 데이터에서 수업료를 추출합니다.
5.  **Notion (Students 조회)**: 해당 학생의 이름 정보를 가져옵니다.
6.  **Code (이름 병합)**: 학생 이름과 이전 단계의 데이터를 병합합니다.
7.  **HTTP (납부회차 조회)**: 등록일원부 DB에서 해당 학생의 기존 납부 기록을 조회합니다.
8.  **Code (납부회차 계산)**: 연도별 납부 회차를 계산하여 다음 회차 번호(`YYNN`)를 생성합니다.
9.  **HTTP (레코드 생성)**: 최종적으로 등록일원부 데이터베이스에 새로운 레코드를 생성합니다.

## ⚙️ 주요 기능
- **자동 종료 감지**: 수업 회차가 모두 완료된 경우에만 자동으로 프로세스가 시작됩니다.
- **지능형 회차 계산**: 연도별로 납부 회차를 추적하여, 기존 기록에 이어지는 다음 회차를 자동으로 부여합니다.
- **데이터 통합**: 수업료, 학생명, 납부일자 등을 자동으로 매핑하여 수기 입력 오류를 방지합니다.

## 🔧 설정 및 유지보수

- **수업료 조회 조건**: `Notion - Class 수업료 조회` 노드에서 Class DB 연결을 확인하세요.
- **납부회차 계산 로직**: `Code - 납부회차 계산` 노드에서 회차 생성 규칙(연도+순번)을 수정할 수 있습니다.
- **등록일원부 DB ID**: `HTTP - 등록일원부 레코드 생성` 노드의 URL 내 DB ID를 최신 상태로 유지해야 합니다.

## 📋 환경 설정
- **Credentials**: 
    - `Notion API`: 워크플로우가 접근하는 모든 Notion 데이터베이스에 대한 읽기/쓰기 권한이 필요합니다.
- **필수 속성**:
    - `Student_Study` DB: `수업완료`(Checkbox), `수업회차`(Multi-select), `수업구분`(Rollup), `Students`(Relation), `Class`(Relation)
    - `등록일원부` DB: `Students`(Relation), `납부회차`(Select) 등

## ⚠️ 주의사항
- Notion API 버전(`2022-06-28`)이 고정되어 있으므로, API 업데이트 시 HTTP 요청 헤더를 확인하십시오.
- `Code` 노드 내의 정규식(`regex`)은 데이터베이스의 속성 이름이나 포맷이 변경될 경우 수정이 필요할 수 있습니다.

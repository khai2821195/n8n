# 등록일원부 자동생성 - 수업종료

본 워크플로우는 Notion의 `Student_Study` 데이터베이스에서 수업이 종료되었을 때, 자동으로 등록일원부 레코드를 생성하여 관리 효율성을 높이는 n8n 자동화 프로세스입니다.

## 📋 워크플로우 개요
- **목적**: 수업 완료 시 자동으로 수업료, 학생 정보, 납부 회차를 계산하여 등록일원부 DB에 레코드 생성
- **핵심 기능**: 
    - 수업 완료 및 중복 생성 방지 체크
    - 수업료 및 학생 이름 데이터 자동 병합
    - 연도별 납부 회차 자동 계산 (예: 2601, 2602...)

## 🔄 전체 흐름
1. **Notion Trigger**: `Student_Study` DB의 변경 사항을 1분 단위로 감시
2. **Code (종료 체크)**: '수업완료' 체크박스 확인 및 '등록일원부생성' 여부 확인(중복 방지)
3. **Notion (Class 조회)**: 해당 수업의 수업료 정보 추출
4. **Code (수업료 추출)**: 수업료 데이터를 이전 데이터와 병합
5. **Notion (Students 조회)**: 학생 이름 정보 추출
6. **Code (이름 병합)**: 학생 이름 데이터를 이전 데이터와 병합
7. **HTTP (납부회차 조회)**: Notion API를 통해 해당 학생의 기존 납부 기록 조회
8. **Code (납부회차 계산)**: 연도별 최신 회차를 계산하여 다음 회차 생성
9. **HTTP (레코드 생성)**: 등록일원부 DB에 최종 데이터 삽입

## ⚙️ 환경 설정 및 크레덴셜

- **데이터베이스 ID**: 
    - `Student_Study` DB ID: `2d0c6a06e17e81188ce1dd2f76406aa1`
    - `등록일원부` DB ID: HTTP 노드의 URL 경로(`databases/{ID}/query`) 내 ID 확인 필요

## 🔧 수정 및 유지보수 가이드
- **수업료 조회 조건**: `Notion - Class 수업료 조회` 노드에서 참조하는 속성명 변경 시 수정
- **납부회차 계산 로직**: `Code - 납부회차 계산` 노드 내 `year` 추출 방식이나 회차 증가 로직 수정 가능
- **DB 연동**: 등록일원부 DB 구조 변경 시 `HTTP - 등록일원부 레코드 생성` 노드의 JSON Body 구조를 업데이트해야 합니다.

## ⚠️ 주의사항
- **중복 방지**: `Code - 종료 체크 및 데이터 추출` 노드에서 `등록일원부생성` 체크박스를 확인하므로, Notion DB 속성에 해당 체크박스가 반드시 존재해야 합니다.

# 등록일원부 자동생성 - 수업종료 워크플로우

본 워크플로우는 Notion의 `Student_Study` 데이터베이스에서 수업이 완료된 시점을 감지하여, 해당 학생의 등록일원부 레코드를 자동으로 생성하는 n8n 자동화 시스템입니다.

---

## 🔄 워크플로우 개요
수업이 종료되었음을 시스템이 감지하면, 수업료 정보와 학생 정보를 조회하고 기존 납부 회차를 계산하여 새로운 등록일원부 데이터를 Notion에 자동으로 기록합니다.

### 전체 흐름
1. **Notion Trigger**: `Student_Study` DB에서 '수업완료' 체크박스가 활성화되면 트리거 발생.
2. **데이터 추출**: 수업 회차와 수업 구분(총 회차)을 비교하여 종료 여부 재검증 및 필요 ID 추출.
3. **수업료 조회**: `Class` DB에서 해당 수업의 수업료 정보를 가져옴.
4. **학생 정보 조회**: `Students` DB에서 학생 이름을 가져옴.
5. **납부 회차 조회**: 기존 등록일원부 DB를 조회하여 해당 학생의 이전 납부 이력을 확인.
6. **납부 회차 계산**: 연도별 순번을 계산하여 새로운 납부 회차(예: 2601) 생성.
7. **레코드 생성**: 최종적으로 등록일원부 DB에 새로운 레코드 생성.

## 🔧 수정 및 유지보수 가이드

*   **수업료 조회 조건**: `Notion - Class 수업료 조회` 노드에서 참조하는 DB 속성 변경 시 수정.
*   **납부회차 계산 로직**: `Code - 납부회차 계산` 노드 내의 연도/순번 포맷팅 로직 수정.
*   **DB ID 변경**: `HTTP - 등록일원부 레코드 생성` 노드의 URL 내 DB ID 값을 최신으로 업데이트.

## 📋 환경 설정 및 주의사항

1. **Credential**: n8n에 Notion API 토큰이 등록되어 있어야 합니다.
2. **Notion 권한**: 해당 워크플로우가 접근하는 모든 Notion 데이터베이스에 연동된 Integration(봇)의 **읽기/쓰기 권한**이 부여되어야 합니다.
3. **데이터 구조**: 
    * `Student_Study` DB에는 '수업완료', '수업회차', '수업구분', 'Students', 'Class' 속성이 필수적으로 존재해야 합니다.
    * `납부회차` 속성은 `연도(2자리) + 순번(2자리)` 형태(예: 2601)로 관리됩니다.

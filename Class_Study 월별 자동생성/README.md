# Class_Study 월별 자동생성 워크플로우

본 워크플로우는 **Math4U** 시스템의 핵심 자동화 프로세스로, 매월 25일에 다음 달의 수업 일정을 자동으로 계산하여 노션(Notion) 데이터베이스에 생성하는 역할을 합니다.

## 📋 워크플로우 개요
- **목적**: 수업 데이터베이스(Class DB)에 등록된 정보를 바탕으로 다음 달 수업 스케줄을 자동 생성하고, 학생별 수업 배정까지 자동으로 처리합니다.
- **실행 주기**: 매월 25일 (Cron: `0 0 25 * *`)
- **주요 기능**:
    - 수업일정 자동 계산 (요일/시간/횟수 기반)
    - 기존 생성된 일정과의 중복 체크
    - Class_Study 및 Student_Study 데이터베이스 자동 업데이트

## 🔄 전체 흐름
1. **Schedule**: 매월 25일 정기 실행
2. **Notion (Class DB 전체 조회)**: 현재 운영 중인 모든 반 정보를 가져옵니다.
3. **Aggregate**: 조회된 반 목록을 하나의 데이터셋으로 병합합니다.
4. **Code (수업일정 계산)**: 수업 구분(횟수)과 요일/시간 설정을 분석하여 다음 달의 구체적인 수업 일정을 계산합니다.
5. **HTTP (다음달 Class_Study 조회)**: 이미 생성된 데이터가 있는지 확인하여 중복 생성을 방지합니다.
6. **If (스킵 체크)**: 중복 여부를 판단하여 신규 생성 여부를 결정합니다.
7. **Notion (생성 및 배정)**:
    - `Class_Study` 페이지 생성
    - `Student_Study` 페이지 생성 (학생별 분리)
    - 배정 확인 처리
8. **Wait**: 노션 API 호출 제한을 고려하여 5초 대기 후 다음 작업 수행

## 🔧 설정 및 수정 가이드

### 1. 환경 설정
- **Notion Credential**: 워크플로우가 접근하는 노션 데이터베이스에 대한 읽기/쓰기 권한이 포함된 API 토큰이 필요합니다.
- **Database ID**: 
    - `Class DB`: 수업 정보가 담긴 데이터베이스 ID
    - `Class_Study DB`: 생성된 수업 일정이 저장될 데이터베이스 ID

### 2. 주요 수정 포인트
- **수업 일정 로직**: `Code - 수업일정 계산` 노드에서 수업 횟수 추출 방식이나 요일 매핑 로직을 수정할 수 있습니다.
- **중복 체크 기준**: `Code - 중복 체크` 노드에서 데이터베이스의 고유 식별자나 날짜 비교 로직을 변경할 수 있습니다.
- **필드 매핑**: `Notion - Student_Study 생성` 노드에서 학생별 데이터 생성 시 필요한 속성(Property)을 수정할 수 있습니다.

## ⚠️ 주의사항
- **API 호출 제한**: 노션 API의 호출 제한(Rate Limit)을 준수하기 위해 `Wait` 노드를 사용하고 있습니다. 대량의 데이터를 처리할 경우 실행 시간이 길어질 수 있습니다.
- **날짜 계산**: `Code` 노드 내에서 KST(한국 표준시) 기준으로 날짜를 계산하고 있으므로, 서버 환경에 따라 시간대 설정 확인이 필요할 수 있습니다.
- **데이터베이스 구조**: 노션 데이터베이스의 속성명(`반명`, `수업구분`, `Students`, `수업 1` 등)이 변경될 경우 `Code` 노드의 코드 수정이 반드시 필요합니다.

# Class_Study 월별 자동생성 워크플로우

본 워크플로우는 매월 25일에 실행되어, 노션(Notion)에 등록된 반(Class) 정보를 바탕으로 다음 달의 수업 일정(`Class_Study`)을 자동으로 생성하고, 학생별 학습 데이터(`Student_Study`)를 연결하는 자동화 시스템입니다.

## ⚙️ 워크플로우 개요
- **목적**: 매월 반복되는 수업 일정 생성 및 학생 배정 업무 자동화
- **실행 주기**: 매월 25일 (Cron: `0 0 25 * *`)
- **주요 기능**:
    - 노션 DB에서 반 정보 및 수업 요일/시간 추출
    - 다음 달 수업 날짜 및 시간 계산 (주차별 자동 넘버링)
    - 기존 데이터와 중복 체크 후 신규 데이터 생성
    - 학생별 학습 레코드 자동 생성 및 배정

## 🔄 전체 흐름
1. **Schedule**: 매월 25일 자동 실행
2. **Notion (Class DB 조회)**: 전체 반 목록을 가져옵니다.
3. **Aggregate**: 조회된 반 목록을 하나로 합칩니다.
4. **Code (수업일정 계산)**: 요일과 시간을 기반으로 다음 달의 모든 수업 일정을 계산하고 넘버링합니다.
5. **Notion (기존 조회)**: 중복 생성을 방지하기 위해 기존 `Class_Study` 데이터를 조회합니다.
6. **Code (중복 체크)**: 이미 생성된 데이터가 있는지 확인합니다.
7. **If (스킵 체크)**: 중복이 아닐 경우에만 다음 단계 진행
    - **Notion (Class_Study 생성)**: 수업 레코드 생성
    - **Code (Students 분리)**: 해당 반의 학생 ID 리스트 추출
    - **Notion (Student_Study 생성)**: 학생별 학습 레코드 생성
    - **Notion (배정확인)**: 관계형 필드 업데이트
8. **Wait**: 5초 대기 후 다음 수업 연결 (API 제한 방지)

## 🔧 수정 및 유지보수
- **수업 일정 로직**: `Code - 수업일정 계산` 노드 내 `getDatesInMonth` 및 `getWeekOfMonth` 함수 수정
- **중복 체크 기준**: `Code - 중복 체크` 노드 수정
- **데이터 필드**: `Notion - Student_Study 생성` 노드에서 노션 속성 매핑 확인 및 수정

## 📋 주의사항
- **필드명 일치**: 노션 데이터베이스의 필드명(`반명`, `Students`, `수업 1~3`, `시간 1~4`, `정규보충`)이 코드 내 정의된 이름과 일치해야 합니다.
- **크레덴셜**: 노션 API 연동을 위한 `Notion API Key`가 n8n에 올바르게 설정되어 있어야 합니다.
- **시간대**: 모든 시간 계산은 한국 시간(KST, UTC+9) 기준으로 처리됩니다.
- **API 제한**: 노션 API의 호출 제한(Rate Limit)을 고려하여 `Wait` 노드를 통해 순차적으로 처리하고 있습니다.

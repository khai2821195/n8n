# Class_Study 월별 자동생성 워크플로우

## 📋 개요
본 워크플로우는 **Math4U** 시스템의 핵심 자동화 프로세스로, 매월 25일에 실행되어 다음 달의 수업 일정(`Class_Study`)을 노션 데이터베이스에 자동으로 생성하고, 학생별 수업 배정을 처리하는 워크플로우입니다.

## 🔄 워크플로우 흐름
1.  **Schedule (매월 25일)**: 매월 25일 정기적으로 워크플로우를 트리거합니다.
2.  **Notion - Class DB 전체 조회**: 수업 정보를 담고 있는 메인 데이터베이스에서 모든 반 정보를 가져옵니다.
3.  **Aggregate**: 조회된 반 목록 데이터를 하나의 배열로 병합합니다.
4.  **Code - 수업일정 계산**: 
    *   각 반의 수업 요일, 시간, 수업 횟수를 분석합니다.
    *   다음 달의 날짜를 계산하여 수업이 있는 날짜와 시간을 매칭합니다.
5.  **HTTP - 다음달 Class_Study 조회**: 이미 생성된 수업 데이터가 있는지 확인하여 중복 생성을 방지합니다.
6.  **Code - 중복 체크**: 계산된 일정과 기존 데이터를 비교하여 생성 대상 여부를 결정합니다.
7.  **If - 스킵 체크**: 중복이 없는 경우에만 다음 단계로 진행합니다.
8.  **Notion - Class_Study 생성**: 계산된 수업 일정을 노션 데이터베이스에 등록합니다.
9.  **Code - Students 분리**: 해당 수업에 배정된 학생 ID 목록을 추출합니다.
10. **Notion - Student_Study 생성**: 학생별 수업 데이터를 생성하여 배정합니다.
11. **Notion - 배정확인**: 최종적으로 배정 상태를 업데이트합니다.
12. **Wait**: 노션 API 제한을 고려하여 5초간 대기 후 다음 수업을 처리합니다.

## ⚙️ 환경 설정 및 크레덴셜
이 워크플로우를 정상적으로 실행하기 위해서는 다음 설정이 필요합니다.

*   **Notion Credentials**: 
    *   `Notion API Key`가 등록되어 있어야 합니다.
    *   워크플로우에서 사용하는 각 데이터베이스(Class DB, Class_Study DB 등)에 해당 API 연동 권한이 부여되어야 합니다.
*   **환경 변수**:
    *   데이터베이스 ID가 노드 내에 올바르게 설정되어 있는지 확인하십시오.

## 🔧 수정 및 유지보수 가이드
*   **수업 일정 계산 로직**: `Code - 수업일정 계산` 노드에서 수업 횟수 계산 방식이나 요일 매핑 로직을 수정할 수 있습니다.
*   **중복 체크 기준**: `Code - 중복 체크` 노드에서 데이터 비교 로직을 변경하여 중복 판단 기준을 조정할 수 있습니다.
*   **데이터 필드 변경**: `Notion - Student_Study 생성` 노드에서 노션 데이터베이스의 속성(Property) 이름이 변경될 경우 해당 노드의 매핑을 업데이트해야 합니다.

## ⚠️ 주의사항
*   **API Rate Limit**: 노션 API 호출 제한을 방지하기 위해 `Wait` 노드가 포함되어 있습니다. 대량의 데이터를 처리할 경우 이 대기 시간을 조정해야 할 수 있습니다.
*   **데이터베이스 ID**: 노션 데이터베이스 구조가 변경되거나 새로운 데이터베이스를 생성할 경우, 각 노드의 `databaseId`를 반드시 최신 값으로 업데이트하십시오.
*   **날짜 계산**: `Code` 노드 내의 날짜 계산 로직은 한국 시간(KST, UTC+9)을 기준으로 작성되어 있습니다. 서버 환경에 따라 시간대 설정 확인이 필요할 수 있습니다.

# Blog Auto Writer 260304

이 워크플로우는 다양한 소스(웹 URL, 노션 페이지, 옵시디언 파일)로부터 정보를 수집하여 자동으로 블로그 포스트를 작성하기 위한 n8n 자동화 파이프라인입니다.

## 📋 워크플로우 개요
사용자가 웹훅을 통해 블로그 주제와 관련 소스 정보를 전달하면, 시스템이 이를 파싱하고 각 소스에서 콘텐츠를 추출하여 블로그 글 작성을 위한 기초 데이터를 준비합니다.

## ⚙️ 동작 방식 (노드 흐름)

1.  **웹훅 트리거 (Webhook Trigger)**: 외부에서 POST 요청을 받아 블로그 주제, 톤앤매너, 타겟 독자, 소스 리스트 등을 수신합니다.
2.  **입력값 파싱 (Set)**: 수신된 데이터를 워크플로우에서 사용하기 적합한 변수로 정규화합니다.
3.  **URL 목록 분리 (Code)**: 입력된 URL, 노션 페이지 ID, 옵시디언 파일 경로를 각각의 배열로 분리합니다.
4.  **데이터 수집 (Branching)**:
    *   **URL 콘텐츠 수집**: Jina.ai를 활용하여 웹 페이지의 텍스트를 읽어옵니다.
    *   **노션 페이지 읽기**: 노션 API를 호출하여 지정된 페이지의 블록 데이터를 가져옵니다.
5.  **결과 정리 (Code)**: 수집된 모든 데이터를 통합하여 블로그 작성을 위한 최종 텍스트 뭉치를 생성합니다.

## 🔑 필수 설정 및 크레덴셜

이 워크플로우를 정상적으로 실행하기 위해서는 다음 항목이 필요합니다.

*   **Notion API**: 노션 페이지를 읽기 위해 `Notion-Version: 2022-06-28` 헤더가 포함된 인증 설정이 필요합니다.
*   **Jina.ai**: 웹 콘텐츠 추출을 위해 `https://r.jina.ai/` 서비스를 사용합니다. (별도의 API 키가 필요할 수 있음)
*   **환경 변수**:
    *   `blog_topic`: 작성할 블로그의 주제
    *   `tone`: 글의 톤 (기본값: '친근한')
    *   `target_reader`: 타겟 독자 (기본값: '학부모')
    *   `quality_loop_max`: 품질 개선 루프 횟수 (기본값: '3')

## 📥 입력 데이터 예시 (JSON)

```json
{
  "blog_topic": "효과적인 자녀 학습법",
  "tone": "전문적인",
  "target_reader": "초등 학부모",
  "url_list": "https://example.com/article1\nhttps://example.com/article2",
  "notion_page_ids": "page_id_1, page_id_2",
  "obsidian_files": "note1.md, note2.md"
}
```

## ⚠️ 주의사항
*   **데이터 제한**: URL 콘텐츠 수집 시 각 소스당 최대 3,000자까지 추출하도록 설정되어 있습니다.
*   **에러 처리**: `Jina URL 읽기` 노드는 요청 실패 시 워크플로우가 중단되지 않도록 `Continue On Fail` 설정이 활성화되어 있습니다.
*   **확장성**: 현재 워크플로우는 데이터 수집 단계까지 구현되어 있습니다. 이후 LLM(OpenAI, Claude 등) 노드를 연결하여 실제 글쓰기 과정을 추가해야 합니다.

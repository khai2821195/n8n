# Blog Auto Writer 260304

이 워크플로우는 다양한 소스(웹 URL, 노션 페이지, 옵시디언 파일)로부터 정보를 수집하여 자동으로 블로그 글을 작성하기 위한 기반을 마련하는 n8n 자동화 파이프라인입니다.

## 📋 워크플로우 개요
- **목적**: 사용자가 제공한 주제와 참고 자료를 바탕으로 블로그 초안을 생성하기 위한 데이터 수집 및 전처리.
- **주요 기능**:
  - 웹훅을 통한 외부 데이터 입력 수신.
  - Jina Reader를 활용한 웹 콘텐츠 텍스트 추출.
  - 노션(Notion) API를 통한 페이지 내용 읽기.
  - 다중 소스 통합 및 데이터 정제.

## ⚙️ 동작 방식 (노드 흐름)

1. **웹훅 트리거 (Webhook)**: `POST` 요청을 통해 블로그 주제, 톤앤매너, 타겟 독자, 참고 URL, 노션 페이지 ID 등의 파라미터를 수신합니다.
2. **입력값 파싱 (Set)**: 수신된 데이터를 워크플로우 내에서 사용할 변수로 할당합니다. (기본값: 톤-친근한, 타겟-학부모, 품질 루프-3회)
3. **URL 목록 분리 (Code)**: 입력된 URL, 노션 ID, 옵시디언 파일 경로를 배열 형태로 파싱합니다.
4. **데이터 수집 단계**:
   - **URL 콘텐츠 수집**: Jina Reader API(`https://r.jina.ai/`)를 사용하여 웹 페이지의 텍스트를 읽어옵니다.
   - **노션 페이지 읽기**: Notion API를 호출하여 지정된 페이지의 블록 데이터를 가져옵니다.
5. **URL 결과 정리 (Code)**: 수집된 모든 웹 콘텐츠를 하나의 텍스트로 병합하여 후속 AI 처리 단계에서 활용할 수 있도록 준비합니다.

## 🔑 필수 설정 및 크레덴셜

이 워크플로우를 정상적으로 실행하려면 다음 항목이 필요합니다.

- **Notion API**:
  - `Notion API Key` (HTTP Header Auth 방식 사용)
  - 노션 페이지에 대한 읽기 권한이 부여된 통합(Integration) 토큰.
- **환경 변수**:
  - 웹훅 경로: `blog-writer`
  - 필요시 Jina Reader API 키 또는 기타 인증 정보가 추가될 수 있습니다.

## ⚠️ 주의사항

- **데이터 제한**: URL 콘텐츠 수집 시 각 소스당 최대 3,000자까지 잘라서 처리하도록 설정되어 있습니다.
- **API 호출**: 노션 API 호출 시 `Notion-Version` 헤더(`2022-06-28`)가 정확히 설정되어 있는지 확인하십시오.
- **확장성**: 현재 워크플로우는 데이터 수집 단계까지 구현되어 있습니다. 실제 블로그 글 작성을 위해서는 이후 단계에 **OpenAI** 또는 **Claude**와 같은 LLM 노드를 연결하여 콘텐츠 생성 로직을 추가해야 합니다.
- **에러 처리**: Jina Reader 호출 시 에러가 발생해도 워크플로우가 중단되지 않고 다음 단계를 진행하도록 설정(`Continue Regular Output`)되어 있습니다.

# Blog Auto Writer 260304

이 워크플로우는 다양한 소스(웹 URL, 노션 페이지, 옵시디언 파일)로부터 정보를 수집하여 자동으로 블로그 글을 작성하기 위한 기반을 마련하는 n8n 자동화 파이프라인입니다.

## 🚀 워크플로우 목적
사용자가 제공한 주제와 타겟 독자, 톤앤매너에 맞춰 여러 외부 소스의 데이터를 통합하고, 이를 기반으로 고품질의 블로그 콘텐츠를 생성하기 위한 전처리 과정을 자동화합니다.

## ⚙️ 동작 방식 (노드 흐름)

1.  **웹훅 트리거 (Webhook Trigger)**: 외부 서비스로부터 블로그 작성 요청(POST)을 수신합니다.
2.  **입력값 파싱 (Set)**: 요청 본문에서 블로그 주제, 톤, 타겟 독자, 소스 목록(URL, 노션 ID 등)을 추출하고 기본값을 설정합니다.
3.  **데이터 분리 (Code)**: 입력받은 문자열 형태의 리스트를 배열(Array) 형태로 변환하여 후속 노드에서 처리하기 쉽게 만듭니다.
4.  **소스별 데이터 수집**:
    *   **URL 콘텐츠 수집**: Jina.ai 리더를 통해 웹 페이지의 텍스트를 추출합니다.
    *   **노션 페이지 읽기**: 노션 API를 호출하여 지정된 페이지의 블록 데이터를 가져옵니다.
5.  **결과 정리**: 수집된 모든 데이터를 하나의 통합된 텍스트 형태로 병합하여 다음 단계(AI 작성 단계)를 준비합니다.

## 🔑 필수 설정 및 크레덴셜

이 워크플로우를 정상적으로 실행하려면 다음 항목이 필요합니다:

*   **Notion API**: 노션 페이지 데이터를 가져오기 위해 `Notion-Version` 헤더가 포함된 HTTP Request 설정이 필요하며, 적절한 API 토큰(Header Auth)이 설정되어야 합니다.
*   **Jina.ai**: 웹 페이지 콘텐츠 추출을 위해 `https://r.jina.ai/` 엔드포인트를 사용합니다. 별도의 API 키가 필요한 경우 HTTP Request 노드 헤더에 추가하십시오.

## 📥 입력 파라미터 (JSON Body 예시)

```json
{
  "blog_topic": "효과적인 자녀 교육법",
  "tone": "친근한",
  "target_reader": "학부모",
  "url_list": "https://example.com/article1\nhttps://example.com/article2",
  "notion_page_ids": "page_id_1, page_id_2",
  "quality_loop_max": "3"
}
```

## ⚠️ 주의사항

*   **데이터 제한**: URL 콘텐츠 수집 시 노드 내에서 3,000자 단위로 텍스트를 자르도록 설정되어 있습니다. 긴 문서를 처리할 경우 이 제한을 조정하십시오.
*   **에러 처리**: `Jina URL 읽기` 노드는 `continueRegularOutput` 옵션이 설정되어 있어, 특정 URL에서 에러가 발생해도 전체 워크플로우가 중단되지 않고 다음 단계를 진행합니다.
*   **확장성**: 현재 워크플로우는 데이터 수집 및 전처리 단계입니다. 실제 블로그 글을 생성하려면 이 뒤에 **AI 모델(OpenAI, Anthropic 등) 노드**를 연결하여 수집된 `url_contents`를 프롬프트에 주입해야 합니다.
